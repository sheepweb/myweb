# 快速测试指南

## 功能1: 备份文件夹结构优化

### 测试步骤

1. **登录管理后台**
   ```
   访问: /admin/login
   ```

2. **配置备份设置**
   - 进入: 设置 → 备份设置
   - 选择备份目标: GitHub 或 Gitee
   - 填写 Token、Owner、Repo
   - 点击"测试连接"确保配置正确
   - 保存设置

3. **创建备份**
   - 点击"立即备份"按钮
   - 等待备份完成

4. **验证文件夹结构**
   - 访问你的 GitHub/Gitee 仓库
   - 检查文件结构是否为: `2026/03/09/backup_db_20260309_xxxxxx.zip`
   - 确认不再是: `2026-03-09/backup_db_20260309_xxxxxx.zip`

### 预期结果

✅ 文件按年/月/日三级文件夹组织
✅ 根目录只有年份文件夹
✅ 便于按时间查找备份文件

## 常见问题排查

### 问题1: 备份文件夹结构没有改变

**可能原因**:
- 代码没有重新编译
- 服务没有重启

**解决方案**:
```bash
# 重新编译
cd /Users/apple/Downloads/goweb
go build -o cboard-go cmd/main.go

# 重启服务
./cboard-go
```

### 问题2: 密码强度提示不显示

**可能原因**:
- 前端代码没有重新构建
- 浏览器缓存

**解决方案**:
```bash
# 重新构建前端
cd frontend
npm run build

# 清除浏览器缓存
# Chrome: Ctrl+Shift+Delete
# 或使用无痕模式测试
```

### 问题3: 密码强度计算不正确

**检查项**:
1. 打开浏览器开发者工具 (F12)
2. 查看 Console 是否有错误
3. 检查 `passwordStrength` 对象的值
4. 确认正则表达式是否正确匹配
