# 功能更新说明

## 2024-03-09 更新

### 1. ✅ 备份文件夹结构优化

**问题**: GitHub/Gitee 仓库根目录文件夹过多，不便于管理。

**解决方案**: 将备份文件按年/月/日的层级结构组织。

**示例**:
```
原来: 2026-03-09/backup_db_20260309_150405.zip
现在: 2026/03/09/backup_db_20260309_150405.zip
```

**文件结构**:
```
GitHub/Gitee 仓库
├── 2026/
│   ├── 01/
│   │   ├── 01/
│   │   │   └── backup_db_20260101_150405.zip
│   │   ├── 02/
│   │   └── ...
│   ├── 02/
│   ├── 03/
│   │   ├── 01/
│   │   ├── 02/
│   │   └── 09/
│   │       └── backup_db_20260309_150405.zip
│   └── ...
└── 2027/
```

**优点**:
- ✅ 更清晰的文件组织结构
- ✅ 便于按年份或月份查找备份
- ✅ 避免根目录文件夹过多
- ✅ 符合常见的归档习惯

**修改文件**: `internal/services/git/git.go`

### 2. ✅ 注册密码实时强度验证

**问题**: 用户注册时不知道密码是否符合要求，导致注册失败率高。

**解决方案**: 添加实时密码强度验证提示，帮助用户创建符合要求的密码。

**功能特性**:

#### 2.1 密码强度等级
- 🔴 **弱**: 满足 ≤2 项要求
- 🟡 **中等**: 满足 3 项要求
- 🔵 **良好**: 满足 4 项要求
- 🟢 **强**: 满足全部 5 项要求

#### 2.2 实时检查项目
1. ✓ 最小长度（默认8个字符，可在后台配置）
2. ✓ 包含小写字母 (a-z)
3. ✓ 包含大写字母 (A-Z)
4. ✓ 包含数字 (0-9)
5. ✓ 包含特殊字符 (!@#$%^&*等)

#### 2.3 视觉反馈
- **进度条**: 根据密码强度显示不同颜色和长度
  - 弱: 红色，20%
  - 中等: 橙色，50%
  - 良好: 蓝色，75%
  - 强: 绿色，100%

- **要求列表**: 每项要求显示状态
  - ✓ 绿色勾号 = 已满足
  - ✗ 红色叉号 = 未满足

- **强度文字**: 显示当前密码强度等级

#### 2.4 响应式设计
- **桌面端**: 要求列表显示为2列
- **手机端**: 要求列表显示为1列，更易阅读

**修改文件**: `frontend/src/views/Register.vue`

**效果预览**:
```
┌─────────────────────────────────────┐
│ 密码: ●●●●●●●●                      │
├─────────────────────────────────────┤
│ ████████████████░░░░░░░░░░░░ 75%   │ ← 进度条
├─────────────────────────────────────┤
│ ✓ 至少 8 个字符    ✓ 包含小写字母  │
│ ✓ 包含大写字母     ✓ 包含数字      │
│ ✗ 包含特殊字符                      │
├─────────────────────────────────────┤
│      密码强度: 良好                 │
└─────────────────────────────────────┘
```

**用户体验优化**:
- ✅ 实时反馈，无需等待提交
- ✅ 清晰的视觉提示
- ✅ 帮助用户快速创建符合要求的密码
- ✅ 减少注册失败率
- ✅ 提高密码安全性

## 技术细节

### 密码强度计算逻辑

```javascript
// 检查各项要求
hasMinLength = password.length >= minPasswordLength
hasLowerCase = /[a-z]/.test(password)
hasUpperCase = /[A-Z]/.test(password)
hasNumber = /\d/.test(password)
hasSpecialChar = /[!@#$%^&*()_+\-=\[\]{}|;:,.<>?]/.test(password)

// 计算满足的条件数量
metCount = 满足的要求数量

// 强度等级
if (metCount <= 2) → 弱 (20%)
if (metCount === 3) → 中等 (50%)
if (metCount === 4) → 良好 (75%)
if (metCount === 5) → 强 (100%)
```

### 样式设计

**颜色方案**:
- 弱: `#f56c6c` (红色)
- 中等: `#e6a23c` (橙色)
- 良好: `#409eff` (蓝色)
- 强: `#67c23a` (绿色)

**动画效果**:
- 进度条: 0.3s 过渡动画
- 要求项: 0.2s 颜色过渡

## 配置说明

### 后台配置项

管理员可以在后台设置中配置以下参数：

1. **最小密码长度** (`min_password_length`)
   - 默认值: 8
   - 范围: 6-50
   - 位置: 设置 → 注册设置

2. **密码复杂度要求**
   - 当前要求: 必须包含字母和数字
   - 建议: 包含大小写字母、数字和特殊字符（3种以上）

## 测试建议

### 测试场景

1. **弱密码测试**
   - 输入: `12345678`
   - 预期: 显示"弱"，只满足长度和数字要求

2. **中等密码测试**
   - 输入: `Abc12345`
   - 预期: 显示"中等"，满足长度、大小写、数字

3. **良好密码测试**
   - 输入: `Abc123456`
   - 预期: 显示"良好"，满足4项要求

4. **强密码测试**
   - 输入: `Abc@12345`
   - 预期: 显示"强"，满足全部5项要求

5. **手机端测试**
   - 检查要求列表是否单列显示
   - 检查触摸操作是否流畅
