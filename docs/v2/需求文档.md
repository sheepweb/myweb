# CBoard v2.0 需求文档

## 1. 项目概述与核心理念

### 1.1 项目定位

CBoard 是一个代理订阅聚合管理平台，核心特点是**不对接服务器**，而是采集/导入节点信息，用户购买套餐后获得聚合订阅地址，通过设备数量限制和订阅重置机制防止滥用。

### 1.2 核心理念

- **资源共享**：将机场资源分享给朋友，通过设备限制和订阅重置防止滥用
- **轻量高效**：单二进制部署，内存占用 35-95 MB，毫秒级启动
- **多数据库支持**：SQLite / MySQL / PostgreSQL 三种可选，开箱即用
- **前后端一体**：go:embed 内嵌前端静态资源，一个文件即可运行

### 1.3 v2.0 重写目标

- 前端从 Element Plus 迁移到 Naive UI，统一设计语言
- 后端代码结构优化，提升可维护性
- 新增 USDT/加密货币支付、卡密兑换
- 新增 Telegram Bot 集成
- 新增在线订阅转换（Surge、Quantumult X、Shadowrocket）
- 新增 CDN/反代集成

## 3. 功能模块详细说明

### 3.1 用户系统

#### 3.1.1 注册与登录

- 邮箱 + 验证码注册（可配置是否需要邀请码）
- 邮箱 + 密码登录（支持 JSON 和 Form 两种格式）
- JWT 双 Token 机制（Access Token + Refresh Token）
- Token 黑名单（登出后立即失效）
- 忘记密码 / 重置密码（邮件验证码）
- Telegram 绑定登录（v2 新增）
- 登录历史记录（IP、UA、GeoIP 定位、设备指纹）
- 异常登录检测与告警（新 IP/新设备自动通知）

验收标准：
- 注册后自动发送验证邮件，验证码 5 分钟有效
- 登录失败 5 次后锁定 15 分钟
- Access Token 有效期可配置（默认 24 小时）
- Refresh Token 有效期可配置（默认 7 天）

#### 3.1.2 用户资料

- 基本信息：用户名、邮箱、昵称、头像
- 偏好设置：主题（亮/暗）、语言、时区
- 通知设置：邮件通知、站内通知、异常登录告警开关
- 隐私设置：数据共享、分析统计开关

#### 3.1.3 用户等级

- 多级等级体系，按累计消费自动升级
- 每级可配置：等级名称、最低消费、折扣率、设备限制、权益说明、图标、颜色
- 等级可设置有效期

#### 3.1.4 邀请系统

- 用户可生成邀请码（可配置最大使用次数、有效期）
- 邀请奖励：邀请人奖励 + 被邀请人奖励（余额形式）
- 可限制仅新用户使用、最低订单金额
- 可绑定指定套餐
- 邀请关系追踪（邀请人、被邀请人、首单、累计消费）
- 佣金日志

#### 3.1.5 余额系统

- 余额充值（通过支付网关）
- 余额消费（购买套餐）
- 余额退款
- 订阅转余额
- 管理员手动调整
- 完整的余额变动日志

### 3.2 订阅系统

#### 3.2.1 订阅管理

- 每个用户一个订阅，包含唯一订阅 URL
- 订阅状态：active / expired / disabled
- 设备限制（由套餐决定，可升级）
- 到期时间追踪，到期自动禁用
- 订阅访问计数（按格式分别统计：Clash、V2Ray、通用）

验收标准：
- 订阅 URL 为随机字符串，不可猜测
- 设备超限时返回明确错误信息
- 到期前 3 天、1 天自动发送提醒通知

#### 3.2.2 订阅格式输出

保留现有格式：
- Clash YAML 格式
- V2Ray Base64 格式
- 通用格式（SSR 等）

v2 新增格式：
- Surge 格式
- Quantumult X 格式
- Shadowrocket 格式
- Stash 格式
- Loon 格式

验收标准：
- 每种格式输出的配置可直接导入对应客户端使用
- 支持在线订阅转换（用户可选择目标格式）

#### 3.2.3 订阅重置

- 用户自助重置（生成新 URL，清除所有设备）
- 管理员重置
- 系统定时重置（可配置）
- 重置历史记录（旧 URL、新 URL、设备数变化、操作人）

#### 3.2.4 订阅转余额

- 用户可将未到期订阅剩余价值转为余额
- 按剩余天数比例计算
- 转换后订阅立即失效

#### 3.2.5 设备升级

- 用户可付费增加设备限制数
- 按月计费，价格可配置
- 升级后立即生效

### 3.3 设备管理

#### 3.3.1 设备识别

- 设备指纹：基于 User-Agent + IP 的 SHA256 哈希
- UA 解析：识别软件名称、版本、操作系统、设备型号、品牌
- 订阅类型识别：Clash / V2Ray / SSR / Surge / Quantumult X 等

#### 3.3.2 设备追踪

- 首次访问时间、最后访问时间、访问次数
- IP 地址记录
- 设备活跃/非活跃状态
- 设备允许/禁止状态（黑白名单）

#### 3.3.3 设备限制

- 按订阅的设备限制数控制
- 超限时拒绝新设备访问
- 用户可手动删除设备释放名额
- 管理员可批量管理设备

验收标准：
- 同一设备（相同指纹）不重复计数
- 设备删除后立即释放名额
- 管理员可查看任意用户的设备列表

### 3.4 节点管理

#### 3.4.1 普通节点

- 节点属性：名称、地区、类型、状态、负载、速度、延迟、上线时间
- 节点排序（order_index）
- 推荐节点标记
- 手动添加 / 自动采集
- 节点配置（Clash YAML 格式存储）

#### 3.4.2 专线节点（Custom Node）

- 独立于普通节点的专线节点管理
- 支持协议：V2Ray、Xray、Shadowsocks、Trojan、Hysteria 等
- 详细配置：域名、端口、UUID、加密方式、传输方式、TLS 等
- 用户分配：多对多关系，指定用户可访问指定专线节点
- 可设置独立到期时间，或跟随用户订阅到期

#### 3.4.3 节点健康检查

- 定时检测节点可用性
- 延迟测试
- 速度测试
- 负载监控
- 自动标记离线节点

#### 3.4.4 节点导入

- 从 Clash 订阅链接批量导入
- 从配置文件导入
- 支持多种协议格式解析
- 自动去重

#### 3.4.5 节点配置自动更新

- 定时从订阅源拉取最新配置
- 自动解析并更新节点列表
- 支持多个订阅源
- 更新日志

### 3.5 套餐与订单

#### 3.5.1 套餐管理

- 套餐属性：名称、描述、价格、时长（天）、设备限制、排序、推荐标记
- 套餐启用/禁用
- 多套餐并存

#### 3.5.2 订单管理

- 订单号（唯一）
- 订单状态：pending → paid → completed / cancelled / refunded
- 关联套餐、用户、优惠券
- 金额计算：原价 - 优惠券折扣 - 等级折扣 = 实付金额
- 订单过期时间（未支付自动取消）
- 支付方式记录
- 管理员可退款

验收标准：
- 订单创建后 30 分钟未支付自动取消
- 支付成功后自动创建/续期订阅
- 退款后自动调整订阅状态

#### 3.5.3 订单统计

- 按时间段统计收入
- 按套餐统计销量
- 按支付方式统计
- 趋势图表

### 3.6 支付系统

#### 3.6.1 支付方式（保留现有 + 新增）

| 支付方式 | 状态 | 说明 |
|----------|------|------|
| 支付宝（直连） | 保留 | 支付宝开放平台 SDK |
| 微信支付 | 保留 | 微信支付 API |
| 易支付（聚合） | 保留 | 支持支付宝/微信/QQ 通道 |
| USDT / 加密货币 | **新增** | 对接 TronLink / 链上监听 |
| 卡密兑换 | **新增** | 管理员生成卡密，用户兑换为余额或套餐 |

#### 3.6.2 支付流程

```
用户选择套餐 → 创建订单 → 选择支付方式 → 跳转/扫码支付
    ↓
支付网关回调 → 验证签名 → 更新订单状态 → 创建/续期订阅
    ↓
通知用户（站内 + 邮件）
```

#### 3.6.4 卡密兑换（新增）

#### 3.6.5 支付配置

## 4. API 设计规范

### 4.1 基础规范

- 基础路径：`/api/v1`
- 认证方式：Bearer Token（JWT）
- 请求格式：JSON（Content-Type: application/json）
- 响应格式：统一 JSON 结构

### 4.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

错误响应：
```json
{
  "code": 40001,
  "message": "参数错误",
  "errors": [
    {"field": "email", "message": "邮箱格式不正确"}
  ]
}
```

### 4.3 错误码规范

| 范围 | 含义 |
|------|------|
| 0 | 成功 |
| 40000-40099 | 参数错误 |
| 40100-40199 | 认证错误 |
| 40300-40399 | 权限错误 |
| 40400-40499 | 资源不存在 |
| 40900-40999 | 冲突（重复） |
| 42900-42999 | 频率限制 |
| 50000-50099 | 服务器内部错误 |

### 4.4 分页规范

请求参数：
- `page`：页码（从 1 开始，默认 1）
- `page_size`：每页数量（默认 20，最大 100）
- `sort`：排序字段
- `order`：排序方向（asc / desc）

响应格式：
```json
{
  "code": 0,
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```
