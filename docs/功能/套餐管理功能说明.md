# 套餐管理功能说明

## 概述

套餐管理功能用于创建和管理用户可购买的订阅套餐。管理员可以设置套餐的价格、时长、设备限制、推荐状态等，用户购买套餐后会自动创建或续费订阅。

## 访问路径

- **管理员后台** → **套餐管理** → **套餐列表**

## 一、创建套餐

### 1. 添加套餐

**步骤：**
1. 点击「添加套餐」按钮
2. 填写套餐信息：
   - **套餐名称**：套餐显示名称（必填，如：基础套餐、高级套餐）
   - **价格**：套餐价格（必填，单位：元，支持小数点后两位）
   - **时长(天)**：套餐有效期（必填，单位：天，如：30 天、90 天）
   - **设备限制**：允许的设备数量（必填，0 表示不限制）
   - **推荐套餐**：是否推荐（开关，推荐套餐会在用户端优先显示）
   - **状态**：启用/禁用（必填，禁用后用户无法购买）
   - **描述**：套餐描述（可选，可自动生成或手动输入）
3. 点击「添加」按钮保存

### 2. 套餐描述自动生成

**功能：**
- 系统会根据价格、时长、设备数量自动生成描述
- 格式示例：「¥100/30天，设备数：3」
- 如果手动编辑描述，系统会使用手动输入的描述

**手动编辑：**
- 在描述框中直接输入自定义描述
- 手动编辑后，系统会使用手动输入的描述，不再自动生成

## 二、编辑套餐

### 1. 修改套餐信息

**步骤：**
1. 在套餐列表中找到要编辑的套餐
2. 点击「编辑」按钮
3. 修改套餐信息（名称、价格、时长、设备限制等）
4. 点击「更新」按钮保存

**注意事项：**
- 修改价格不会影响已购买的订单
- 修改时长不会影响已生效的订阅
- 修改设备限制会影响新购买的套餐
- 禁用套餐后，用户无法购买，但不影响已购买的订阅

### 2. 套餐状态管理

**启用/禁用：**
- **启用**：套餐正常显示，用户可以购买
- **禁用**：套餐不显示，用户无法购买，但不影响已购买的订阅

**使用场景：**
- **禁用**：套餐下架、临时停售、需要修改价格时
- **启用**：套餐重新上架、恢复正常销售时

## 三、删除套餐

### 1. 删除单个套餐

**步骤：**
1. 在套餐列表中找到要删除的套餐
2. 点击「删除」按钮
3. 确认删除操作

**注意事项：**
- 删除套餐不会影响已购买的订单和订阅
- 删除操作不可恢复
- 建议先禁用套餐，确认无影响后再删除

### 2. 删除限制

**安全限制：**
- 删除套餐不会影响已购买的订单
- 删除套餐不会影响已生效的订阅
- 删除套餐后，用户无法再购买该套餐

## 四、套餐搜索和筛选

### 1. 搜索套餐

**搜索方式：**
- **套餐名称**：输入套餐名称关键词搜索
- **状态筛选**：按启用/禁用状态筛选

**搜索功能：**
- 支持模糊搜索
- 支持按状态筛选
- 支持组合搜索（名称 + 状态）

### 2. 筛选选项

- **全部状态**：显示所有套餐
- **启用**：只显示启用的套餐
- **禁用**：只显示禁用的套餐

## 五、套餐配置说明

### 1. 价格设置

**格式：**
- 支持小数点后两位（如：99.99 元）
- 最小值为 0
- 建议设置合理的价格，避免设置过低或过高

**注意事项：**
- 价格修改不会影响已购买的订单
- 用户购买时按当前价格计算
- 用户等级折扣会在购买时自动应用

### 2. 时长设置

**单位：**
- 以「天」为单位
- 最小值为 1 天
- 常见设置：30 天、90 天、180 天、365 天

**计算方式：**
- 用户购买后，订阅到期时间 = 当前时间 + 套餐时长
- 如果用户已有订阅，购买后会自动续费（延长到期时间）

### 3. 设备限制设置

**设置说明：**
- **0**：不限制设备数量
- **1-999**：限制设备数量上限
- 用户购买套餐后，订阅的设备限制会更新为该值

**注意事项：**
- 设备限制修改不会影响已生效的订阅
- 用户购买新套餐时，设备限制会更新为新套餐的值
- 如果用户当前设备数超过新限制，不会自动删除设备，但无法添加新设备

### 4. 推荐套餐设置

**功能：**
- 推荐套餐会在用户端优先显示
- 用户端会突出显示推荐套餐
- 可以设置多个推荐套餐

**使用场景：**
- 主推套餐：设置为主推套餐，吸引用户购买
- 促销套餐：促销活动时设置为推荐套餐
- 高价值套餐：高价值套餐设置为推荐套餐

## 六、套餐与订阅的关系

### 1. 购买流程

```
用户选择套餐
 ↓
创建订单（包含套餐信息）
 ↓
支付订单
 ↓
订单支付成功
 ↓
创建或续费订阅（应用套餐的时长和设备限制）
```

### 2. 套餐信息应用

**应用到订阅：**
- **时长**：订阅到期时间 = 当前时间 + 套餐时长
- **设备限制**：订阅的设备限制 = 套餐的设备限制
- **价格**：订单价格 = 套餐价格（可能应用用户等级折扣）

### 3. 续费逻辑

**续费方式：**
- 如果用户已有订阅，购买套餐后会延长到期时间
- 到期时间 = 原到期时间 + 套餐时长（如果原到期时间未过期）
- 到期时间 = 当前时间 + 套餐时长（如果原到期时间已过期）

## 七、使用场景

### 场景 1：创建基础套餐

```
1. 点击「添加套餐」
2. 填写信息：
   - 套餐名称：基础套餐
   - 价格：29.99
   - 时长：30 天
   - 设备限制：3
   - 推荐套餐：是
   - 状态：启用
3. 点击「添加」保存
```

### 场景 2：创建促销套餐

```
1. 创建新套餐（如：限时优惠套餐）
2. 设置较低价格
3. 设置为推荐套餐
4. 促销结束后，禁用套餐
```

### 场景 3：调整套餐价格

```
1. 找到要调整的套餐
2. 点击「编辑」
3. 修改价格
4. 点击「更新」保存
5. 新价格立即生效，已购买的订单不受影响
```

## 八、注意事项

1. **价格修改**：修改价格不会影响已购买的订单和订阅
2. **时长修改**：修改时长不会影响已生效的订阅
3. **设备限制**：修改设备限制不会影响已生效的订阅
4. **删除套餐**：删除套餐不会影响已购买的订单和订阅
5. **推荐套餐**：可以设置多个推荐套餐
6. **套餐描述**：系统会自动生成描述，也可以手动编辑

## 九、常见问题

### Q1: 修改套餐价格后，已购买的订单会受影响吗？

**答：**
- 不会。修改套餐价格只影响新购买的订单
- 已购买的订单价格不会改变
- 已生效的订阅不受影响

### Q2: 如何下架套餐？

**方法：**
- 在套餐列表中，将套餐状态改为「禁用」
- 禁用后，用户无法购买该套餐
- 已购买的订阅不受影响

### Q3: 套餐删除后，已购买的订阅会受影响吗？

**答：**
- 不会。删除套餐不会影响已购买的订单和订阅
- 已生效的订阅会继续正常使用
- 只是用户无法再购买该套餐

### Q4: 如何设置多个推荐套餐？

**方法：**
- 在创建或编辑套餐时，开启「推荐套餐」开关
- 可以设置多个套餐为推荐套餐
- 推荐套餐会在用户端优先显示

### Q5: 套餐的设备限制如何生效？

**说明：**
- 用户购买套餐后，订阅的设备限制会更新为套餐的设备限制
- 如果用户当前设备数超过新限制，不会自动删除设备
- 但用户无法添加新设备，直到设备数低于限制
