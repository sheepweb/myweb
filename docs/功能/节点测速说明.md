# 节点测速说明

## 概述

节点测速功能用于测试节点的连通性和延迟，帮助管理员了解节点状态，及时发现故障节点。系统支持单个节点测速和批量节点测速，测试结果会更新节点的状态和延迟信息。

## 访问路径

- **管理员后台** → **节点管理** → **节点管理**（普通节点测速）
- **管理员后台** → **节点管理** → **专线节点管理**（专线节点测速）

## 一、单个节点测速

### 1. 普通节点测速

**步骤：**
1. 在节点管理页面，找到要测试的节点
2. 点击节点操作列中的「测试」按钮（或「测速」按钮）
3. 系统开始测试节点连通性和延迟
4. 测试完成后显示结果：
   - **状态**：online（在线）/ offline（离线）/ timeout（超时）
   - **延迟**：延迟时间（毫秒）

**测试结果说明：**
- **online**：节点在线，延迟正常（延迟 < 最大允许延迟）
- **offline**：节点离线，无法连接
- **timeout**：节点超时，延迟过高或响应超时（延迟 > 最大允许延迟 或 超过测试超时时间）

### 2. 专线节点测速

**步骤：**
1. 在专线节点管理页面，找到要测试的节点
2. 点击节点操作列中的「测试」按钮
3. 系统开始测试节点连通性和延迟
4. 测试完成后显示结果：
   - **状态**：active（活跃）/ inactive（非活跃）/ error（错误）
   - **延迟**：延迟时间（毫秒）

**测试结果说明：**
- **active**：节点活跃，延迟正常
- **inactive**：节点非活跃，可能离线或延迟过高
- **error**：节点错误，配置错误或无法连接

## 二、批量节点测速

### 1. 批量测试普通节点

**步骤：**
1. 在节点管理页面，勾选要测试的节点（可以多选）
2. 点击「批量测试」按钮（或「批量测速」按钮）
3. 系统会依次测试所有选中的节点
4. 测试完成后显示每个节点的测试结果

**测试结果：**
- 每个节点都会显示测试结果
- 节点状态和延迟信息会自动更新
- 可以在节点列表中查看更新后的状态

### 2. 批量测试专线节点

**步骤：**
1. 在专线节点管理页面，勾选要测试的节点（可以多选）
2. 点击「批量测速」按钮
3. 系统会依次测试所有选中的节点
4. 测试完成后显示每个节点的测试结果

**测试结果：**
- 每个节点都会显示测试结果
- 节点状态和延迟信息会自动更新

## 三、测速原理

### 1. 测试方式

系统支持两种测试方式：

**方式一：HTTP 测试（如果配置了测速网站 URL）**
- 通过 HTTP 请求测试节点延迟
- 使用配置的测速网站 URL（如 `https://ping.pe`）
- 更准确地反映实际使用体验

**方式二：TCP 连接测试（如果未配置测速网站 URL）**
- 直接测试 TCP 连接延迟
- 测试速度更快
- 可能不如 HTTP 测试准确

### 2. 测试参数

测试使用以下参数（可在「节点健康检查设置」中配置）：

- **最大允许延迟**：默认 3000 毫秒（3 秒）
  - 延迟超过此值，节点会被标记为超时
- **测试超时时间**：默认 5 秒
  - 单个节点测试的最大等待时间
- **测速网站 URL**：默认 `https://ping.pe`
  - 用于 HTTP 测试的网页地址

### 3. 测试流程

```
1. 系统发起测试请求
 ↓
2. 建立到节点的连接
 ↓
3. 如果配置了测速网站 URL：
   - 通过节点代理访问测速网站
   - 记录响应时间
 ↓
4. 如果未配置测速网站 URL：
   - 直接测试 TCP 连接
   - 记录连接延迟
 ↓
5. 对比延迟与阈值
   - 延迟 < 最大允许延迟：标记为 online/active
   - 延迟 > 最大允许延迟：标记为 timeout/inactive
   - 连接失败：标记为 offline/error
 ↓
6. 更新节点状态和延迟信息
 ↓
7. 返回测试结果
```

## 四、测速结果说明

### 1. 延迟范围

- **优秀**：< 100 毫秒（适合低延迟需求）
- **良好**：100-300 毫秒（适合大多数场景）
- **一般**：300-1000 毫秒（可用，但体验一般）
- **较差**：1000-3000 毫秒（延迟较高）
- **超时**：> 3000 毫秒 或 连接超时（不可用）

### 2. 状态说明

**普通节点状态：**
- **online**：在线，延迟正常，可以使用
- **offline**：离线，无法连接，不可使用
- **timeout**：超时，延迟过高或响应超时，不可使用

**专线节点状态：**
- **active**：活跃，延迟正常，可以使用
- **inactive**：非活跃，可能离线或延迟过高，不可使用
- **error**：错误，配置错误或无法连接，不可使用

## 五、使用场景

### 场景 1：定期检查节点状态

**步骤：**
1. 定期（如每天）批量测试所有节点
2. 查看测试结果，找出离线或超时的节点
3. 及时处理故障节点（修复或删除）

### 场景 2：添加新节点后测试

**步骤：**
1. 添加新节点后，立即测试节点
2. 确认节点状态为 online/active
3. 确认延迟在可接受范围内

### 场景 3：节点故障排查

**步骤：**
1. 用户反馈节点无法使用
2. 管理员测试该节点
3. 查看测试结果（状态、延迟）
4. 根据结果判断问题：
   - 如果 offline/error：节点配置错误或服务器故障
   - 如果 timeout/inactive：延迟过高或网络问题
5. 采取相应措施（修复配置、更换节点等）

## 六、注意事项

1. **测试时间**：批量测试可能需要较长时间，请耐心等待
2. **网络环境**：测试结果受服务器网络环境影响
3. **测试频率**：不建议过于频繁地测试，避免对节点服务器造成压力
4. **测试结果**：测试结果仅供参考，实际使用体验可能有所不同
5. **自动测试**：系统会定期自动测试节点（见「节点健康检查设置」）

## 七、常见问题

### Q1: 测试失败？

**检查项：**
- 节点配置是否正确
- 服务器地址是否可访问
- 端口是否正确
- 网络连接是否正常
- 测速网站 URL 是否可访问（如果使用 HTTP 测试）

**解决方法：**
- 检查节点配置
- 验证服务器连接
- 检查网络环境
- 查看系统日志中的错误信息

### Q2: 测试结果显示延迟很高？

**可能原因：**
- 节点服务器距离较远
- 网络环境较差
- 节点服务器负载过高
- 测试时网络波动

**解决方法：**
- 多次测试，取平均值
- 检查节点服务器状态
- 考虑更换节点或优化网络

### Q3: 批量测试耗时过长？

**可能原因：**
- 测试节点数量过多
- 测试超时时间设置过长
- 部分节点响应慢

**解决方法：**
- 分批测试节点
- 适当缩短测试超时时间
- 优先测试重要节点

### Q4: 测试结果不准确？

**可能原因：**
- 测试方式选择不当
- 网络环境不稳定
- 节点服务器临时故障

**解决方法：**
- 配置测速网站 URL，使用 HTTP 测试
- 多次测试，取平均值
- 结合用户反馈判断节点状态

## 八、测速配置建议

### 1. 测试间隔

- **手动测试**：根据需要随时测试
- **自动测试**：建议 30-60 分钟（见「节点健康检查设置」）

### 2. 延迟阈值

- **严格**：1000-2000 毫秒（适合对速度要求高的场景）
- **标准**：3000 毫秒（推荐，平衡速度和可用性）
- **宽松**：5000 毫秒或更长（适合网络环境较差的场景）

### 3. 测试超时时间

- **快速**：3-5 秒（适合节点数量多、需要快速检查的场景）
- **标准**：5-10 秒（推荐，平衡检查速度和准确性）
- **宽松**：10-15 秒（适合网络环境较差的场景）
