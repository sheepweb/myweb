## 场景1：赠送余额活动

### 步骤1：创建活动
```bash
curl -X POST http://localhost:8080/api/v1/admin/promotions \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "新用户注册礼包",
    "type": "new_user",
    "discount_type": "balance",
    "discount_value": 10.00,
    "min_amount": 0,
    "start_time": "2026-04-28T00:00:00Z",
    "end_time": "2026-05-28T23:59:59Z",
    "is_active": true,
    "description": "新用户注册即送10元余额"
  }'
```

### 步骤2：用户参与活动
```bash
curl -X POST http://localhost:8080/api/v1/promotions/1/participate \
  -H "Authorization: Bearer <user_token>"
```

### 步骤3：验证结果
```sql
-- 检查用户余额是否增加
SELECT id, username, balance FROM users WHERE id = <user_id>;

-- 检查参与记录
SELECT * FROM promotion_participations WHERE user_id = <user_id>;

-- 检查余额日志
SELECT * FROM balance_logs WHERE user_id = <user_id> ORDER BY created_at DESC LIMIT 1;
```

**预期结果**：
- ✅ 用户余额增加10元
- ✅ 参与记录状态为 "completed"
- ✅ 余额日志记录正确

## 场景3：折扣券活动（自动应用）

### 步骤1：创建百分比折扣活动
```bash
curl -X POST http://localhost:8080/api/v1/admin/promotions \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "限时8折优惠",
    "type": "flash_sale",

    "discount_value": 20,
    "min_amount": 50,
    "max_discount": 100,
    "start_time": "2026-04-28T00:00:00Z",
    "end_time": "2026-05-28T23:59:59Z",
    "is_active": true,
    "description": "全场8折，最高优惠100元"
  }'
```

### 步骤2：用户参与活动
```bash
curl -X POST http://localhost:8080/api/v1/promotions/3/participate \
  -H "Authorization: Bearer <user_token>"
```

### 步骤3：查看可用折扣券
```bash
curl -X GET http://localhost:8080/api/v1/promotions/available-discounts \
  -H "Authorization: Bearer <user_token>"
```

### 步骤4：创建订单（不使用优惠券）
```bash
curl -X POST http://localhost:8080/api/v1/orders \
  -H "Authorization: Bearer <user_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "package_id": 1,
    "duration_months": 1,
    "payment_method": "yipay_alipay"
  }'
```

### 步骤5：验证折扣应用
```sql
-- 检查订单折扣
SELECT 
  id, 
  order_no, 
  amount, 
  discount_amount, 
  final_amount 
FROM orders 
WHERE user_id = <user_id> 
ORDER BY created_at DESC 
LIMIT 1;

-- 检查参与记录（应该关联到订单）
SELECT 
  id, 
  promotion_id, 
  order_id, 
  reward_type, 
  reward_value, 
  status 
FROM promotion_participations 
WHERE user_id = <user_id> AND promotion_id = 3;
```

**预期结果**：
- ✅ 订单 discount_amount 包含营销活动折扣
- ✅ 参与记录 order_id 关联到订单
- ✅ 参与记录状态为 "pending"（支付前）

### 步骤6：完成支付
模拟支付回调或使用实际支付

### 步骤7：验证支付后状态
```sql
-- 检查参与记录状态
SELECT 
  id, 
  promotion_id, 
  order_id, 
  status, 
  applied_at 
FROM promotion_participations 
WHERE user_id = <user_id> AND promotion_id = 3;
```

**预期结果**：
- ✅ 参与记录状态更新为 "completed"
- ✅ applied_at 时间已设置

## 场景5：优惠券与营销活动折扣互斥

### 前提条件
- 用户已参与营销活动，有可用折扣券
- 用户有可用优惠券

### 步骤1：使用优惠券创建订单
```bash
curl -X POST http://localhost:8080/api/v1/orders \
  -H "Authorization: Bearer <user_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "package_id": 1,
    "duration_months": 1,
    "coupon_code": "TEST2024",
    "payment_method": "yipay_alipay"
  }'
```

### 步骤2：验证折扣应用
```sql
-- 检查订单
SELECT 
  id, 
  order_no, 
  amount, 
  discount_amount, 
  final_amount,
  coupon_id 
FROM orders 
WHERE user_id = <user_id> 
ORDER BY created_at DESC 
LIMIT 1;

-- 检查营销活动参与记录
SELECT 
  id, 
  order_id, 
  status 
FROM promotion_participations 
WHERE user_id = <user_id> AND status = 'pending';
```

**预期结果**：
- ✅ 订单使用优惠券折扣（coupon_id 不为空）
- ✅ 营销活动折扣券未被使用（order_id 为空，status 仍为 pending）
- ✅ 折扣券可以在下次购买时使用

## 数据库验证查询

```sql
-- 查看所有活动
SELECT id, name, type, discount_type, discount_value, is_active 
FROM promotions 
ORDER BY created_at DESC;

-- 查看所有参与记录
SELECT 
  pp.id,
  pp.promotion_id,
  p.name as promotion_name,
  pp.user_id,
  u.username,
  pp.order_id,
  pp.reward_type,
  pp.reward_value,
  pp.status,
  pp.applied_at,
  pp.expire_at
FROM promotion_participations pp
LEFT JOIN promotions p ON pp.promotion_id = p.id
LEFT JOIN users u ON pp.user_id = u.id
ORDER BY pp.created_at DESC;

-- 查看订单折扣详情
SELECT 
  o.id,
  o.order_no,
  o.user_id,
  o.package_id,
  o.amount,
  o.discount_amount,
  o.final_amount,
  o.status,
  pp.promotion_id,
  p.name as promotion_name
FROM orders o
LEFT JOIN promotion_participations pp ON o.id = pp.order_id
LEFT JOIN promotions p ON pp.promotion_id = p.id
WHERE o.user_id = <user_id>
ORDER BY o.created_at DESC;
```
