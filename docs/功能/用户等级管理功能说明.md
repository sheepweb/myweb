# 用户等级管理功能说明

## 概述

用户等级管理功能用于创建和管理用户等级系统，支持自动升级和等级折扣。用户根据累计消费自动升级等级，不同等级享受不同的套餐折扣优惠。

## 访问路径

- **管理员后台** → **用户管理** → **用户等级管理**

## 一、创建用户等级

### 1. 添加等级

**步骤：**
1. 点击「添加等级」按钮
2. 填写等级信息：
   - **等级名称**：等级显示名称（必填，如：VIP、超级VIP、钻石会员）
   - **等级排序**：等级排序数字（必填，数字越小等级越高，1 为最高等级）
   - **最低消费**：累计消费达到此金额可升级（必填，单位：元）
   - **折扣率**：等级折扣率（可选，0.9 表示 9 折，1.0 表示无折扣）
   - **等级颜色**：用于前端显示等级的颜色（可选）
   - **图标 URL**：等级图标 URL（可选）
   - **权益说明**：等级权益说明，JSON 格式（可选）
   - **是否启用**：启用/禁用等级（开关）
3. 点击「保存」按钮

### 2. 等级排序说明

**排序规则：**
- 数字越小，等级越高（1 为最高等级）
- 建议从低到高设置：10、5、2、1
- 排序用于确定等级高低关系

**示例：**
- 普通会员：排序 10（最低等级）
- VIP 会员：排序 5（中等等级）
- 超级 VIP：排序 2（高等级）
- 钻石会员：排序 1（最高等级）

### 3. 最低消费设置

**设置说明：**
- 用户累计消费达到此金额可升级到此等级
- 单位：元，支持小数点后两位
- 建议从低到高递增设置（如：0 元、100 元、500 元）

**升级逻辑：**
- 用户累计消费达到等级要求时，系统会自动升级
- 升级后用户享受该等级的折扣优惠
- 升级是自动的，无需手动操作

### 4. 折扣率设置

**设置说明：**
- **0.9**：9 折（100 元套餐只需支付 90 元）
- **0.95**：95 折（100 元套餐只需支付 95 元）
- **1.0**：无折扣（100 元套餐支付 100 元）
- 范围：0.1 - 1.0，支持小数点后两位

**折扣应用：**
- 用户购买套餐时自动应用等级折扣
- 折扣金额会显示在订单中
- 用户可以看到节省的金额

## 二、编辑用户等级

### 1. 修改等级信息

**步骤：**
1. 在等级列表中找到要编辑的等级
2. 点击「编辑」按钮
3. 修改等级信息（名称、排序、最低消费、折扣率等）
4. 点击「保存」按钮

**注意事项：**
- 修改最低消费会影响用户的升级条件
- 修改折扣率会影响用户的购买价格
- 修改排序会影响等级高低关系
- 禁用等级后，用户无法升级到此等级

### 2. 等级状态管理

**启用/禁用：**
- **启用**：等级正常使用，用户可以达到此等级
- **禁用**：等级停用，用户无法升级到此等级，但不影响已升级的用户

**使用场景：**
- **禁用**：等级需要调整、临时停用等级时
- **启用**：等级恢复正常使用

## 三、删除用户等级

### 1. 删除等级

**步骤：**
1. 在等级列表中找到要删除的等级
2. 点击「删除」按钮
3. 确认删除操作

**注意事项：**
- 删除等级不会影响已升级到此等级的用户
- 删除操作不可恢复
- 建议先禁用等级，确认无影响后再删除

### 2. 删除限制

**安全限制：**
- 删除等级不会影响用户的当前等级
- 删除等级不会影响用户的累计消费
- 删除等级不会影响用户的折扣优惠

## 四、等级搜索和筛选

### 1. 搜索等级

**搜索方式：**
- **状态筛选**：按启用/禁用状态筛选

**筛选选项：**
- **全部**：显示所有等级
- **启用**：只显示启用的等级
- **禁用**：只显示禁用的等级

## 五、等级系统工作原理

### 1. 自动升级机制

**升级条件：**
- 用户累计消费达到等级的最低消费要求
- 系统自动检查用户累计消费
- 自动升级到符合条件的最高等级

**升级时机：**
- 用户购买套餐后，系统检查累计消费
- 如果达到升级条件，自动升级等级
- 升级后立即享受新等级的折扣优惠

### 2. 折扣应用机制

**应用时机：**
- 用户创建订单时
- 系统自动应用用户当前等级的折扣
- 折扣金额显示在订单中

**折扣计算：**
- 订单价格 = 套餐价格 × 折扣率
- 折扣金额 = 套餐价格 - 订单价格
- 用户可以看到节省的金额

### 3. 折扣叠加

**叠加规则：**
- 等级折扣和优惠券折扣可以叠加使用
- 叠加顺序：先应用等级折扣，再应用优惠券折扣
- 用户享受双重优惠

**示例：**
- 套餐价格：100 元
- 等级折扣：9 折（VIP）
- 优惠券折扣：8 折
- 最终价格：100 × 0.9 × 0.8 = 72 元

## 六、用户端显示

### 1. 等级显示位置

**用户仪表盘：**
- 在首页顶部显示当前等级（带颜色标识）
- 显示等级名称和颜色
- 显示升级进度条

**升级进度条：**
- 显示距离下一级还需消费的金额
- 显示进度百分比
- 显示当前累计消费和下一级要求

**订单支付：**
- 创建订单时自动应用等级折扣
- 显示折扣金额和节省金额
- 提醒用户如何获取优惠价格

### 2. 等级标识

**颜色标识：**
- 每个等级可以设置不同的颜色
- 颜色用于前端显示等级标识
- 帮助用户识别等级高低

**图标标识：**
- 每个等级可以设置图标 URL
- 图标用于前端显示等级标识
- 增强视觉效果

## 七、配置建议

### 1. 等级设置建议

**等级数量：**
- 建议设置 3-5 个等级
- 等级过多可能让用户感到困惑
- 等级过少可能缺乏激励效果

**等级命名：**
- 使用清晰的等级名称（如：VIP、超级VIP、钻石会员）
- 避免使用过于复杂的名称
- 让用户容易理解等级高低

**消费门槛：**
- 建议从低到高递增设置（如：0 元、100 元、500 元、1000 元）
- 门槛不宜过高，避免用户难以达到
- 门槛不宜过低，避免失去激励效果

### 2. 折扣设置建议

**折扣幅度：**
- 建议设置合理的折扣幅度（如：9 折、95 折）
- 折扣过低可能影响利润
- 折扣过高可能缺乏激励效果

**折扣递增：**
- 建议等级越高，折扣越大
- 鼓励用户消费升级
- 增强用户粘性

## 八、使用场景

### 场景 1：创建基础等级系统

```
1. 创建普通会员：排序 10，最低消费 0 元，折扣 1.0（无折扣）
2. 创建 VIP 会员：排序 5，最低消费 100 元，折扣 0.95（95 折）
3. 创建超级 VIP：排序 2，最低消费 500 元，折扣 0.9（9 折）
4. 启用所有等级
```

### 场景 2：调整等级折扣

```
1. 找到要调整的等级（如：VIP 会员）
2. 点击「编辑」
3. 修改折扣率（如：从 0.95 改为 0.9）
4. 点击「保存」
5. 已升级到此等级的用户立即享受新折扣
```

### 场景 3：添加新等级

```
1. 点击「添加等级」
2. 设置等级名称：钻石会员
3. 设置排序：1（最高等级）
4. 设置最低消费：1000 元
5. 设置折扣率：0.85（85 折）
6. 启用等级
7. 用户累计消费达到 1000 元时自动升级
```

## 九、注意事项

1. **等级排序**：数字越小等级越高（1 为最高等级）
2. **最低消费**：用户累计消费达到此金额可升级
3. **折扣率**：0.9 表示 9 折，1.0 表示无折扣
4. **自动升级**：用户累计消费达到等级要求时自动升级
5. **折扣应用**：用户购买套餐时自动应用等级折扣
6. **折扣叠加**：等级折扣和优惠券折扣可以叠加使用

## 十、常见问题

### Q1: 用户如何升级等级？

**答：**
- 用户累计消费达到等级的最低消费要求时，系统会自动升级
- 升级是自动的，无需手动操作
- 用户可以在个人中心查看升级进度

### Q2: 修改等级折扣会影响已升级的用户吗？

**答：**
- 会。修改等级折扣后，已升级到此等级的用户立即享受新折扣
- 用户购买套餐时会自动应用新折扣
- 建议谨慎修改折扣率

### Q3: 如何查看用户的当前等级？

**方法：**
- 在用户列表中，查看用户的等级信息
- 在用户详情中，查看用户的等级和累计消费
- 用户可以在个人中心查看自己的等级

### Q4: 等级折扣和优惠券折扣可以叠加吗？

**答：**
- 可以。等级折扣和优惠券折扣可以叠加使用
- 叠加顺序：先应用等级折扣，再应用优惠券折扣
- 用户享受双重优惠

### Q5: 删除等级会影响已升级的用户吗？

**答：**
- 不会。删除等级不会影响已升级到此等级的用户
- 用户的当前等级不受影响
- 但用户无法再升级到此等级
