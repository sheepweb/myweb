# 专线节点管理说明

## 概述

专线节点是为特定用户分配的专属节点，提供更高的性能和更好的服务质量。专线节点可以分配给单个或多个用户，支持到期时间管理、流量控制等功能。

## 访问路径

- **管理员后台** → **节点管理** → **专线节点管理**

## 一、创建专线节点

### 1. 链接导入方式（推荐）

**步骤：**
1. 点击「创建节点」按钮
2. 选择「链接导入」标签页
3. 在文本框中粘贴节点链接（每行一个，支持批量导入）
4. 支持的协议：vmess、vless、trojan、ss、hysteria2、tuic、naive 等
5. 点击「批量导入」按钮

**示例：**
```
vmess://eyJ2IjoiMiIsInBzIjoi5Lit5Zu9IiwiaG9zdCI6IjEuMi4zLjQiLCJwb3J0Ijo0NDMsInR5cGUiOiJub25lIiwiaWQiOiJ1dWlkLTEyMzQ1NiIsImFpZCI6MCwic25pIjoiIiwiYWxwbiI6IiJ9
vless://uuid-123456@1.2.3.4:443?encryption=none&security=tls#香港专线
```

**解析预览：**
- 可以点击「仅解析预览」按钮查看第一个链接的解析结果
- 显示节点名称、服务器地址、端口等信息

### 2. 手动填写方式

**步骤：**
1. 点击「创建节点」按钮
2. 选择「手动填写」标签页
3. 填写节点信息：
   - **名称**：节点名称（必填）
   - **显示名称**：节点显示名称（可选，用于用户端显示）
   - **协议**：节点协议类型（必填，vmess、vless、trojan、ss 等）
   - **配置**：节点配置 JSON（必填）
   - **域名**：节点域名（可选）
   - **端口**：节点端口（可选，从配置中提取）
   - **到期时间**：节点到期时间（可选）
   - **跟随用户到期**：是否跟随用户订阅到期时间（开关）
4. 点击「保存节点」按钮

**配置 JSON 格式：**
```json
{
  "type": "vmess",
  "server": "1.2.3.4",
  "port": 443,
  "uuid": "uuid-12345678-1234-1234-1234-123456789abc",
  "network": "ws",
  "wsPath": "/path",
  "security": "tls"
}
```

## 二、编辑专线节点

### 1. 编辑节点信息

**步骤：**
1. 在专线节点列表中找到要编辑的节点
2. 点击「编辑」按钮
3. 修改节点信息（名称、显示名称、到期时间等）
4. 点击「保存」按钮

**注意事项：**
- 编辑时不能修改协议和配置（如需修改，删除后重新创建）
- 修改到期时间会影响已分配的用户
- 「跟随用户到期」开启时，节点到期时间会跟随用户订阅到期时间

### 2. 查看节点链接

**步骤：**
1. 在节点列表中点击「链接」按钮
2. 系统会显示节点的完整链接
3. 可以复制链接用于测试或分享

## 三、删除专线节点

### 1. 单个删除

**步骤：**
1. 在节点列表中找到要删除的节点
2. 点击「删除」按钮
3. 确认删除操作
4. 系统会删除节点并取消所有用户的分配

**注意事项：**
- 删除节点会同时取消所有用户的分配
- 删除操作不可恢复
- 删除后，已分配的用户将无法使用该节点

### 2. 批量删除

**步骤：**
1. 在节点列表中勾选要删除的节点（可以多选）
2. 点击「批量删除」按钮
3. 确认删除操作
4. 系统会批量删除选中的节点

**注意事项：**
- 批量删除会同时取消所有用户的分配
- 删除操作不可恢复
- 建议先取消分配，再删除节点

## 四、节点分配

### 1. 单个节点分配

**步骤：**
1. 在节点列表中点击「分配」按钮
2. 在分配对话框中搜索用户（输入用户名或邮箱）
3. 选择要分配的用户（可以多选）
4. 配置分配选项（可选）：
   - **订阅类型**：`both`（普通+专线）、`special_only`（仅专线）
   - **到期时间**：节点到期时间（可选）
5. 点击「确认分配」按钮

**分配结果：**
- 节点已分配给选中的用户
- 用户可以在订阅中看到专线节点
- 专线节点会显示在订阅列表的最前面

### 2. 批量节点分配

**步骤：**
1. 在节点列表中勾选要分配的节点（可以多选）
2. 点击「批量分配」按钮
3. 在分配对话框中搜索用户
4. 选择要分配的用户（可以多选）
5. 配置分配选项（可选）
6. 点击「确认分配」按钮

**分配结果：**
- 所有选中的节点都会分配给选中的用户
- 每个用户都会收到所有选中的节点

### 3. 从用户详情分配

**步骤：**
1. 在用户列表中点击用户，查看用户详情
2. 切换到「专线节点」标签页
3. 点击「分配节点」按钮
4. 搜索并选择要分配的节点
5. 配置分配选项（可选）
6. 点击「确认分配」按钮

## 五、取消分配

### 1. 从节点详情取消分配

**步骤：**
1. 在节点列表中点击「分配」按钮，查看已分配的用户
2. 在已分配用户列表中找到要取消的用户
3. 点击「移除」按钮
4. 确认取消分配操作

**结果：**
- 节点已从该用户取消分配
- 用户将无法再使用该节点
- 节点可以重新分配给其他用户

### 2. 从用户详情取消分配

**步骤：**
1. 在用户列表中点击用户，查看用户详情
2. 切换到「专线节点」标签页
3. 找到要取消分配的节点
4. 点击「取消分配」按钮
5. 确认取消分配操作

**结果：**
- 节点已从该用户取消分配
- 用户将无法再使用该节点

## 六、节点测速

### 1. 单个节点测速

**步骤：**
1. 在节点列表中点击「测试」按钮
2. 系统会测试节点的连通性和延迟
3. 显示测试结果：状态（active/inactive/error）、延迟（毫秒）

**测试结果说明：**
- **active**：节点在线，延迟正常
- **inactive**：节点离线或延迟过高
- **error**：节点配置错误或无法连接

### 2. 批量节点测速

**步骤：**
1. 在节点列表中勾选要测试的节点（可以多选）
2. 点击「批量测速」按钮
3. 系统会依次测试所有选中的节点
4. 显示每个节点的测试结果

**注意事项：**
- 批量测试可能需要较长时间
- 测试结果会更新节点的状态和延迟信息
- 测试失败不会影响节点配置

## 七、节点状态管理

### 1. 激活/禁用节点

**步骤：**
1. 在节点列表中，使用「激活」开关
2. 开启：节点启用，分配给用户后可以使用
3. 关闭：节点禁用，分配给用户后无法使用

**使用场景：**
- **禁用**：节点故障或维护时，临时禁用节点
- **激活**：节点恢复正常后，重新激活节点

### 2. 节点状态说明

- **active**：活跃，节点正常
- **inactive**：非活跃，节点可能离线
- **error**：错误，节点配置错误或无法连接

## 八、节点到期管理

### 1. 设置到期时间

**方式一：创建时设置**
- 在创建节点时，填写「到期时间」
- 节点到期后，用户将无法使用

**方式二：编辑时设置**
- 在编辑节点时，修改「到期时间」
- 修改后立即生效

### 2. 跟随用户到期

**功能：**
- 开启「跟随用户到期」开关
- 节点到期时间会自动跟随用户订阅到期时间
- 用户订阅续费时，节点也会自动续期

**使用场景：**
- 适合专线节点与用户订阅绑定
- 简化到期时间管理

## 九、配置示例

### 示例 1：创建并分配专线节点

```
1. 创建节点（链接导入）
   - 粘贴节点链接
   - 点击「批量导入」
   
2. 分配节点
   - 点击「分配」按钮
   - 搜索用户「user@example.com」
   - 选择用户
   - 订阅类型：both
   - 点击「确认分配」
   
3. 测试节点
   - 点击「测试」按钮
   - 确认节点状态为 active
```

### 示例 2：批量分配多个节点

```
1. 创建多个节点（批量导入）
   - 粘贴多个节点链接（每行一个）
   - 点击「批量导入」
   
2. 批量分配
   - 勾选所有要分配的节点
   - 点击「批量分配」
   - 搜索并选择用户
   - 配置分配选项
   - 点击「确认分配」
```

## 十、注意事项

1. **节点 ID**：专线节点的 ID 大于 1000000，用于区分普通节点
2. **分配关系**：一个节点可以分配给多个用户，一个用户可以有多个专线节点
3. **订阅类型**：用户订阅类型为 `special_only` 时，只显示专线节点
4. **到期时间**：节点到期后，用户将无法使用，需要续期或重新分配
5. **删除节点**：删除节点会取消所有用户的分配，请谨慎操作

## 十一、常见问题

### Q1: 如何查看节点已分配给哪些用户？

**方法：**
1. 在节点列表中点击「分配」按钮
2. 查看「已分配用户」列表
3. 可以看到所有已分配的用户信息

### Q2: 节点分配后用户看不到？

**检查项：**
- 节点是否已激活（`is_active = true`）
- 用户订阅类型是否正确（`both` 或 `special_only`）
- 节点是否已到期
- 用户订阅是否已过期

**解决方法：**
- 检查节点状态和用户订阅状态
- 确认分配关系是否正确
- 查看系统日志中的错误信息

### Q3: 如何批量取消分配？

**方法：**
- 目前需要逐个取消分配
- 或删除节点（会取消所有用户的分配）

### Q4: 节点测速失败？

**检查项：**
- 节点配置是否正确
- 服务器地址是否可访问
- 端口是否正确
- 网络连接是否正常

**解决方法：**
- 检查节点配置
- 验证服务器连接
- 查看系统日志中的错误信息
