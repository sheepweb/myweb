# 优惠券折扣计算说明

## 折扣计算顺序

优惠券系统采用**分层折扣**机制，按以下顺序计算：

1. **基础价格**：套餐原价 × 购买月数
2. **用户等级折扣**：如果用户有等级，先应用等级折扣率
3. **优惠券折扣**：在等级折扣后的金额上再应用优惠券
4. **余额抵扣**：最后使用账户余额抵扣

## 优惠券类型

### 1. 折扣券 (discount)

按百分比折扣，例如 20% 折扣。

**计算公式**：
```
折扣金额 = (应用等级折扣后的金额) × (折扣率 / 100)
```

**限制**：
- `max_discount`: 最大折扣金额上限
- `min_amount`: 最低订单金额要求

**示例**：
- 订单原价：100元
- 用户等级折扣：10%（折后90元）
- 优惠券：20%折扣，最大折扣40元
- 计算：90 × 20% = 18元
- 最终金额：90 - 18 = 72元

### 2. 固定金额券 (fixed)

直接减免固定金额，例如减30元。

**计算公式**：
```
折扣金额 = min(优惠券面值, 应用等级折扣后的金额)
```

**示例**：
- 订单原价：100元
- 用户等级折扣：10%（折后90元）
- 优惠券：减30元
- 最终金额：90 - 30 = 60元

### 3. 赠送天数券 (free_days)

赠送额外的服务天数（暂未实现）。

## 优惠券验证规则

### 1. 基本验证
- ✅ 优惠券状态必须为 `active`
- ✅ 当前时间必须在有效期内（`valid_from` ~ `valid_until`）
- ✅ 优惠券总量未用完（如果设置了 `total_quantity`）

### 2. 用户限制
- ✅ 用户使用次数未超过限制（`max_uses_per_user`）
- ✅ 每个用户的使用记录独立计算

### 3. 金额限制
- ✅ 订单金额（应用等级折扣后）必须满足最低金额要求（`min_amount`）
- ✅ 折扣金额不能超过订单金额

## 完整计算示例

### 示例1：折扣券 + 用户等级 + 余额

**条件**：
- 套餐原价：100元/月，购买1个月
- 用户等级：VIP（9折）
- 优惠券：20%折扣，最大折扣40元
- 账户余额：50元，选择使用余额

**计算过程**：
```
1. 基础价格 = 100元
2. 用户等级折扣 = 100 × (1 - 0.9) = 10元
   等级折扣后 = 100 - 10 = 90元
3. 优惠券折扣 = 90 × 20% = 18元
   优惠券折扣后 = 90 - 18 = 72元
4. 余额抵扣 = min(50, 72) = 50元
   最终需支付 = 72 - 50 = 22元
```

**结果**：
- 总优惠：10 + 18 + 50 = 78元
- 实际支付：22元

### 示例2：固定金额券 + 最低金额限制

**条件**：
- 套餐原价：50元/月
- 优惠券：减30元，最低金额100元

**结果**：
- ❌ 不满足使用条件（订单金额50元 < 最低金额100元）

### 示例3：折扣券 + 最大折扣限制

**条件**：
- 套餐原价：200元
- 优惠券：50%折扣，最大折扣40元

**计算过程**：
```
1. 基础价格 = 200元
2. 优惠券折扣 = 200 × 50% = 100元
3. 受限于最大折扣 = min(100, 40) = 40元
4. 最终金额 = 200 - 40 = 160元
```

## API 接口

### 验证优惠券

**接口**：`POST /api/v1/coupons/verify`

**请求**：
```json
{
  "code": "SAVE20",
  "amount": 100.00,
  "package_id": 1
}
```

**响应**：
```json
{
  "code": 200,
  "message": "",
  "data": {
    "coupon": { ... },
    "base_amount": 100.00,
    "level_discount_amount": 10.00,
    "coupon_discount_amount": 18.00,
    "total_discount_amount": 28.00,
    "final_amount": 72.00
  }
}
```

### 创建订单时使用优惠券

**接口**：`POST /api/v1/orders`

**请求**：
```json
{
  "package_id": 1,
  "coupon_code": "SAVE20",
  "payment_method": "alipay",
  "use_balance": true,
  "duration_months": 1
}
```

## 注意事项

1. **优惠券在等级折扣之后应用**：确保用户等级折扣先生效，优惠券在折扣后的金额上再计算
2. **防止并发超卖**：使用数据库行锁（FOR UPDATE）防止优惠券被超量使用
3. **原子操作**：优惠券使用记录和计数器更新在同一事务中完成
4. **金额精度**：所有金额计算保留2位小数，使用 `RoundFloat` 函数
5. **验证一致性**：`VerifyCoupon` 和 `CreateOrder` 使用相同的计算逻辑
