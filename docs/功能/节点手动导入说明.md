# 节点手动导入说明

## 概述

节点手动导入功能允许管理员通过多种方式添加节点到系统，包括链接导入、手动填写、Clash 配置导入、文件导入等。手动导入的节点会被标记为「手动添加」，不会被定时采集任务清理。

## 访问路径

- **管理员后台** → **节点管理** → **节点管理** → **添加节点**

## 一、链接导入（推荐）

### 1. 单个链接导入

**步骤：**
1. 点击「添加节点」按钮
2. 选择「链接导入」标签页
3. 在文本框中粘贴节点链接（支持 vmess、vless、trojan、ss、ssr、hysteria、tuic、naive、anytls 等）
4. 点击「批量导入」按钮

**支持的链接格式：**
- `vmess://...`
- `vless://uuid@server:port?params#name`
- `trojan://password@server:port?params#name`
- `ss://base64(method:password)@server:port#name`
- `ssr://base64(...)`
- `hysteria://server:port?params#name`
- `tuic://uuid:password@server:port?params#name`
- `naive+https://username:password@server:port#name`
- `anytls://server:port?params#name`

**示例：**
```
vmess://eyJ2IjoiMiIsInBzIjoi5Lit5Zu9IiwiaG9zdCI6IjEuMi4zLjQiLCJwb3J0Ijo0NDMsInR5cGUiOiJub25lIiwiaWQiOiJ1dWlkLTEyMzQ1NiIsImFpZCI6MCwic25pIjoiIiwiYWxwbiI6IiJ9
```

### 2. 批量链接导入

**步骤：**
1. 在「链接导入」文本框中，每行粘贴一个节点链接
2. 可以粘贴多个链接（每行一个）
3. 点击「批量导入」按钮
4. 系统会显示导入结果：成功数、跳过数（已存在的节点）

**示例：**
```
vmess://eyJ2IjoiMiIsInBzIjoi5Lit5Zu9IiwiaG9zdCI6IjEuMi4zLjQiLCJwb3J0Ijo0NDMsInR5cGUiOiJub25lIiwiaWQiOiJ1dWlkLTEyMzQ1NiIsImFpZCI6MCwic25pIjoiIiwiYWxwbiI6IiJ9
vless://uuid-123456@1.2.3.4:443?encryption=none&security=tls#香港节点
trojan://password123@1.2.3.4:443?allowInsecure=0#日本节点
```

**导入结果：**
- **成功**：新节点已添加
- **跳过**：节点已存在（基于 Type:Server:Port 去重）

### 3. 解析预览

**功能：**
- 在导入前，可以点击「仅解析预览」按钮
- 系统会解析第一个链接，显示节点名称、服务器地址、端口等信息
- 用于验证链接格式是否正确

## 二、手动填写

### 1. 手动创建节点

**步骤：**
1. 点击「添加节点」按钮
2. 选择「手动填写」标签页
3. 填写节点信息：
   - **名称**：节点显示名称（必填）
   - **地区**：节点所在地区（必填，如：香港、日本、美国）
   - **类型**：节点协议类型（必填，VMess、VLESS、Trojan、SS）
   - **配置**：节点配置 JSON（必填）
   - **推荐**：是否推荐节点（开关）
   - **激活**：是否启用节点（开关，默认开启）
4. 点击「保存节点」按钮

### 2. 配置 JSON 格式

**VMess 示例：**
```json
{
  "server": "1.2.3.4",
  "port": 443,
  "uuid": "uuid-12345678-1234-1234-1234-123456789abc",
  "alterId": 0,
  "network": "ws",
  "wsPath": "/path",
  "tls": true
}
```

**VLESS 示例：**
```json
{
  "server": "1.2.3.4",
  "port": 443,
  "uuid": "uuid-12345678-1234-1234-1234-123456789abc",
  "encryption": "none",
  "network": "ws",
  "wsPath": "/path",
  "security": "tls"
}
```

**Trojan 示例：**
```json
{
  "server": "1.2.3.4",
  "port": 443,
  "password": "your-password",
  "sni": "example.com",
  "allowInsecure": false
}
```

**SS 示例：**
```json
{
  "server": "1.2.3.4",
  "port": 443,
  "method": "aes-256-gcm",
  "password": "your-password"
}
```

## 三、Clash 配置导入

### 1. 从 Clash 配置导入

**步骤：**
1. 在节点管理页面，找到「导入 Clash 配置」功能（如果有）
2. 粘贴完整的 Clash YAML 配置内容
3. 系统会自动解析配置中的节点链接
4. 导入所有解析到的节点

**说明：**
- 系统会从 Clash 配置中提取所有节点链接
- 支持各种协议格式
- 导入的节点会被标记为「手动添加」

### 2. 从文件导入

**步骤：**
1. 将 Clash 配置文件（`clash.yaml`）上传到服务器
2. 文件路径：`./uploads/config/clash.yaml`
3. 通过 API 或后台功能导入文件
4. 系统读取文件内容并解析节点

**文件格式：**
- 必须是有效的 YAML 格式
- 包含 `proxies` 数组，每个元素包含节点信息
- 节点信息中包含 `server`、`port`、`type` 等字段

## 四、导入结果说明

### 1. 导入成功

- 节点已添加到系统
- 节点状态为「在线」（online）
- 节点来源标记为「手动添加」（`IsManual = true`）

### 2. 导入跳过

- 节点已存在（基于 Type:Server:Port 去重）
- 系统会更新现有节点的信息（名称、地区、配置等）
- 不会创建重复节点

### 3. 导入失败

**常见原因：**
- 链接格式不正确
- 链接已损坏或无效
- 配置 JSON 格式错误
- 必填字段缺失

**解决方法：**
- 检查链接格式是否正确
- 验证链接是否有效
- 检查配置 JSON 格式
- 查看系统日志中的错误信息

## 五、节点去重机制

### 1. 去重规则

系统基于以下组合判断节点是否重复：
- **Type:Server:Port**：节点类型、服务器地址、端口
- 如果存在 UUID，还会检查 UUID
- 如果存在 Password，还会检查 Password

### 2. 去重时机

- **导入时**：导入链接时自动去重
- **采集时**：采集节点时自动去重
- **手动添加时**：手动填写时也会检查是否重复

### 3. 去重处理

- **已存在**：更新现有节点信息，不创建新节点
- **不存在**：创建新节点

## 六、注意事项

1. **手动节点不会被清理**：手动添加的节点（`IsManual = true`）不会被定时采集任务清理
2. **链接格式**：确保链接格式正确，否则无法解析
3. **配置 JSON**：手动填写时，配置 JSON 必须格式正确
4. **地区识别**：系统会自动识别地区，也可以手动设置
5. **节点状态**：导入后节点状态为「在线」，系统会定期检查节点健康状态

## 七、常见问题

### Q1: 链接导入失败？

**检查项：**
- 链接格式是否正确
- 链接是否完整（没有截断）
- 链接是否有效（可以正常使用）

**解决方法：**
- 使用「仅解析预览」功能验证链接格式
- 检查链接是否完整
- 尝试使用其他格式的链接

### Q2: 批量导入时部分节点失败？

**说明：**
- 批量导入时，系统会逐个处理每个链接
- 失败的链接不会影响其他链接的导入
- 查看导入结果中的成功数和跳过数

**解决方法：**
- 检查失败的链接格式
- 单独导入失败的链接
- 查看系统日志中的错误信息

### Q3: 如何编辑已导入的节点？

**方法：**
1. 在节点列表中找到要编辑的节点
2. 点击「编辑」按钮
3. 修改节点信息
4. 点击「保存」按钮

### Q4: 如何删除导入的节点？

**方法：**
- **单个删除**：在节点列表中点击「删除」按钮
- **批量删除**：选中多个节点，点击「批量删除」按钮

**注意事项：**
- 删除节点会从订阅中移除
- 删除操作不可恢复
- 专线节点（ID > 1000000）删除需要特殊处理
