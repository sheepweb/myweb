## 3. 营销活动功能

### ✅ 新增功能：完整实现

**功能概述**：
管理员创建营销活动后，用户可以参与活动并获得奖励。支持三种奖励类型：
1. **赠送余额** (balance)
2. **赠送订阅天数** (free_days)
3. **折扣券** (discount: percentage/fixed)

### 3.1 管理员创建活动

**接口**：`POST /api/v1/admin/promotions`

**请求示例**：
```json
{
  "name": "新用户注册礼包",
  "type": "new_user",
  "discount_type": "balance",
  "discount_value": 10.00,
  "min_amount": 0,
  "max_discount": 0,
  "package_ids": null,
  "start_time": "2026-04-28T00:00:00Z",
  "end_time": "2026-05-28T23:59:59Z",
  "is_active": true,
  "description": "新用户注册即送10元余额"
}
```

**活动类型**：
- `flash_sale` - 限时抢购
- `new_user` - 新用户专享
- `recall` - 召回活动
- `member_day` - 会员日

**奖励类型**：
- `balance` - 赠送余额（单位：元）
- `free_days` - 赠送天数（单位：天）
- `percentage` - 百分比折扣（如：20 表示 20% 折扣）
- `fixed` - 固定金额折扣（单位：元）

### 3.2 用户参与活动

**接口**：`POST /api/v1/promotions/:id/participate`

**请求**：无需请求体，活动ID在URL中

**响应示例**：
```json
{
  "code": 200,
  "message": "恭喜！您已成功领取 ¥10.00 余额",
  "data": null
}
```

**验证逻辑**：
1. ✅ 检查用户是否登录
2. ✅ 检查活动是否存在
3. ✅ 检查活动是否有效（is_active = true）
4. ✅ 检查活动时间范围（start_time ~ end_time）
5. ✅ 检查用户是否已参与（防止重复领取）
6. ✅ 根据奖励类型应用奖励

**奖励应用逻辑**：

#### 赠送余额 (balance)
```sql
UPDATE users SET balance = balance + discount_value WHERE id = user_id
```
- ✅ 直接增加用户余额
- ✅ 记录余额变动日志
- ✅ 立即生效

#### 赠送天数 (free_days)
```sql
UPDATE subscriptions 
SET expire_time = expire_time + INTERVAL discount_value DAY 
WHERE user_id = user_id
```
- ✅ 延长订阅到期时间
- ✅ 如果订阅已过期，从当前时间开始计算
- ✅ 需要用户已有订阅，否则返回错误

#### 折扣券 (discount)
- ✅ 创建参与记录，状态为 `pending`
- ✅ 设置过期时间为活动结束时间
- ✅ 下次购买时自动应用（需要集成到订单创建流程）

### 3.4 查询可用折扣券

**接口**：`GET /api/v1/promotions/available-discounts`

**响应**：返回用户未使用的折扣券列表（status = "pending"）

### 3.5 自动应用折扣券

**函数**：`applyPromotionDiscount(userID, packageID, baseAmount)`

**集成位置**：在 `CreateOrder` 函数中调用，在优惠券折扣之后、余额扣除之前应用

**代码位置**：
- `internal/services/order/order.go:121-133` - 调用折扣应用逻辑
- `internal/services/order/order.go:1044-1100` - 折扣计算函数
- `internal/services/order/order.go:224-231` - 关联订单到参与记录
- `internal/services/order/order.go:455-467` - 支付成功后更新状态

**逻辑**：
1. ✅ 查找用户可用的折扣券（status = "pending", 未过期）
2. ✅ 检查活动是否适用于该套餐（package_ids）
3. ✅ 检查是否满足最低金额（min_amount）
4. ✅ 计算折扣金额（百分比或固定金额）
5. ✅ 在创建订单时关联参与记录到订单
6. ✅ 订单支付成功后，更新参与记录状态为 "completed"

**折扣应用顺序**：
1. 基础价格（套餐价格 × 月数）
2. 用户等级折扣
3. 优惠券折扣（如果有）
4. **营销活动折扣**（如果没有优惠券）
5. 余额抵扣

**注意**：营销活动折扣和优惠券折扣互斥，优先使用优惠券。

## 数据库迁移

### 新增表：promotion_participations

```sql
CREATE TABLE promotion_participations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    promotion_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    order_id INTEGER,
    reward_type VARCHAR(50) NOT NULL,
    reward_value DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    applied_at DATETIME,
    expire_at DATETIME,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (promotion_id) REFERENCES promotions(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (order_id) REFERENCES orders(id)
);
```

**执行迁移**：
```bash
sqlite3 data/cboard.db < scripts/migrations/add_promotion_participations.sql
```
