# 订阅重置功能说明

## 概述

订阅重置功能用于重置用户的订阅地址，清除所有设备记录，生成新的订阅链接。重置后，用户需要重新添加设备才能使用订阅服务。系统支持用户自己重置和管理员重置两种方式。

## 访问路径

- **用户端** → **订阅管理** → **重置订阅**
- **管理员后台** → **订阅管理** → **订阅列表** → **重置**按钮

## 一、用户自己重置订阅

### 1. 重置订阅地址

**步骤：**
1. 用户登录系统，进入「订阅管理」页面
2. 找到「重置订阅」按钮或选项
3. 点击「重置订阅」按钮
4. 确认重置操作
5. 系统生成新的订阅地址，清除所有设备记录

**重置结果：**
- 订阅地址已更新（生成新的订阅 URL）
- 所有设备记录已清除
- 设备数量重置为 0
- 用户需要重新添加设备才能使用订阅

### 2. 重置原因

**常见原因：**
- 账户可能泄露，需要更换订阅地址
- 设备数量异常，需要清理设备
- 订阅地址丢失，需要重新获取
- 其他安全考虑

### 3. 重置影响

**对用户的影响：**
- 所有设备无法继续使用订阅
- 需要重新添加设备
- 订阅到期时间不受影响
- 订阅设备限制不受影响

## 二、管理员重置订阅

### 1. 单个订阅重置

**步骤：**
1. 在订阅列表中找到要重置的订阅
2. 点击「重置」按钮
3. 确认重置操作
4. 系统重置订阅地址，清除所有设备记录

**重置结果：**
- 订阅地址已更新
- 所有设备记录已清除
- 设备数量重置为 0
- 重置记录会保存到订阅重置历史

### 2. 批量重置订阅

**步骤：**
1. 在订阅列表中勾选要重置的订阅（可以多选）
2. 点击「批量重置」按钮
3. 确认批量重置操作
4. 系统批量重置选中的订阅

**批量重置结果：**
- 所有选中的订阅地址已更新
- 所有选中的订阅的设备记录已清除
- 显示重置成功和失败的数量

### 3. 从用户详情重置

**步骤：**
1. 在用户列表中点击用户，查看用户详情
2. 在用户详情中找到订阅信息
3. 点击「重置订阅」按钮
4. 确认重置操作

**重置结果：**
- 该用户的所有订阅地址已重置
- 所有设备记录已清除

## 三、订阅重置记录

### 1. 查看重置记录

**位置：**
- 在用户详情对话框中，切换到「订阅重置记录」标签页
- 显示该用户的所有订阅重置记录

**记录信息：**
- **重置时间**：重置操作的时间
- **重置类型**：手动重置、自动重置、管理员重置、系统重置
- **操作人**：执行重置操作的用户（用户自己、管理员、系统）
- **重置前设备数**：重置前的设备数量
- **重置后设备数**：重置后的设备数量（通常为 0）
- **重置原因**：重置的原因说明

### 2. 重置类型说明

**手动重置：**
- 用户自己主动重置订阅
- 重置原因：用户主动重置订阅地址

**自动重置：**
- 系统自动重置订阅（如果有自动重置规则）

**管理员重置：**
- 管理员手动重置用户订阅
- 重置原因：管理员重置用户订阅地址

**系统重置：**
- 系统自动重置订阅（如：安全检测、异常处理等）

## 四、订阅延长功能

### 1. 延长订阅时间

**功能：**
- 管理员可以手动延长用户的订阅到期时间
- 延长天数可以自定义
- 延长后订阅到期时间会自动更新

**步骤：**
1. 在订阅列表中找到要延长的订阅
2. 点击「延长」按钮（如果有）
3. 输入要延长的天数
4. 确认延长操作

**延长结果：**
- 订阅到期时间已更新
- 到期时间 = 原到期时间 + 延长天数（如果原到期时间未过期）
- 到期时间 = 当前时间 + 延长天数（如果原到期时间已过期）

### 2. 延长使用场景

**常见场景：**
- 用户投诉订阅时间计算错误，需要补偿
- 促销活动，赠送订阅时长
- 用户服务问题，需要补偿订阅时间
- 其他需要延长订阅的情况

## 五、发送订阅邮件

### 1. 发送订阅邮件给用户

**功能：**
- 管理员可以手动发送订阅邮件给用户
- 邮件包含订阅地址和相关信息
- 用于用户订阅地址丢失或需要重新获取时

**步骤：**
1. 在订阅列表中找到要发送邮件的订阅
2. 点击「发送邮件」按钮
3. 系统发送订阅邮件到用户邮箱

**邮件内容：**
- 订阅地址（通用订阅、Clash 订阅等）
- 订阅到期时间
- 设备限制信息
- 其他订阅相关信息

### 2. 发送限制

**注意事项：**
- 需要配置邮件服务器才能发送邮件
- 邮件发送可能需要一些时间
- 避免频繁发送，避免被标记为垃圾邮件

## 六、清理用户设备

### 1. 清理用户所有设备

**功能：**
- 管理员可以清理用户的所有设备记录
- 清理后设备数量重置为 0
- 用户需要重新添加设备

**步骤：**
1. 在用户详情中，找到「清理设备」按钮
2. 点击「清理设备」按钮
3. 确认清理操作

**清理结果：**
- 用户的所有设备记录已删除
- 设备数量重置为 0
- 订阅地址不受影响

### 2. 清理与重置的区别

**清理设备：**
- 只清除设备记录
- 订阅地址不变
- 设备数量重置为 0

**重置订阅：**
- 清除设备记录
- 生成新的订阅地址
- 设备数量重置为 0

**选择建议：**
- **清理设备**：只清理设备，订阅地址不变
- **重置订阅**：清理设备并更换订阅地址，更安全

## 七、使用场景

### 场景 1：用户账户泄露

```
1. 用户发现账户可能泄露
2. 用户自己重置订阅地址
3. 或联系管理员重置订阅地址
4. 重新添加设备，使用新订阅地址
```

### 场景 2：设备数量异常

```
1. 管理员发现用户设备数量异常（如：10/3）
2. 管理员重置用户订阅
3. 清除所有设备记录
4. 用户重新添加设备
```

### 场景 3：订阅时间补偿

```
1. 用户投诉订阅时间计算错误
2. 管理员延长订阅时间（如：延长 7 天）
3. 订阅到期时间自动更新
4. 用户继续使用订阅
```

### 场景 4：批量重置订阅

```
1. 发现批量账户泄露或异常
2. 管理员批量选择异常订阅
3. 批量重置订阅地址
4. 通知用户重新添加设备
```

## 八、注意事项

1. **重置不可恢复**：重置订阅后，设备记录无法恢复
2. **订阅地址变化**：重置后会生成新的订阅地址，旧地址失效
3. **设备需要重新添加**：重置后，用户需要重新添加设备
4. **订阅时间不受影响**：重置不会影响订阅到期时间
5. **设备限制不受影响**：重置不会影响订阅的设备限制
6. **重置记录保存**：所有重置操作都会记录到重置历史中

## 九、常见问题

### Q1: 重置订阅会影响订阅到期时间吗？

**答：**
- 不会。重置订阅只影响订阅地址和设备记录
- 订阅到期时间不受影响
- 订阅设备限制不受影响

### Q2: 重置订阅后，用户需要做什么？

**答：**
- 用户需要重新添加设备
- 用户需要使用新的订阅地址
- 用户可以在订阅管理页面查看新的订阅地址

### Q3: 如何查看订阅重置历史？

**方法：**
- 在用户详情对话框中，切换到「订阅重置记录」标签页
- 查看该用户的所有重置记录
- 包括重置时间、类型、操作人、原因等

### Q4: 批量重置订阅会失败吗？

**可能原因：**
- 用户信息不存在或已被删除
- 订阅信息不存在
- 系统错误

**处理方式：**
- 系统会显示重置成功和失败的数量
- 失败的订阅不会影响其他订阅的重置
- 可以单独处理失败的订阅

### Q5: 重置订阅和清理设备有什么区别？

**区别：**
- **重置订阅**：清除设备记录 + 生成新订阅地址
- **清理设备**：只清除设备记录，订阅地址不变

**选择建议：**
- 如果只是设备数量异常，可以选择清理设备
- 如果账户可能泄露，建议重置订阅
