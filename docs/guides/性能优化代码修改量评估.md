## 二、后端优化修改量

### 1. Redis 缓存扩展 🟠 中等
**修改文件**: 10-15 个（各个 handler）
**修改行数**: 每个文件约 10-20 行
**修改时间**: 每个接口 15-20 分钟

**修改示例**:
```go
// 在每个 handler 中添加缓存逻辑（约 15 行）
func GetUser(c *gin.Context) {
    id := c.Param("id")
    cacheKey := "user:" + id

    // 查缓存（新增 5 行）
    if cached, err := redis.Get(cacheKey); err == nil {
        c.JSON(200, cached)
        return
    }

    // 查数据库（原有代码）
    var user User
    db.First(&user, id)

    // 写缓存（新增 2 行）
    redis.Set(cacheKey, user, 5*time.Minute)

    c.JSON(200, user)
}
```

**修改难度**: ⭐⭐ 简单
**风险**: 低
**收益**: 接口响应速度提升 90-95%

**修改时间**: 每个接口 15 分钟

```go
// 在每个列表接口添加分页（约 10 行）
func GetUsers(c *gin.Context) {
    page, _ := strconv.Atoi(c.DefaultQuery("page", "1"))
    pageSize, _ := strconv.Atoi(c.DefaultQuery("pageSize", "20"))

    var users []User
    var total int64

    offset := (page - 1) * pageSize

    db.Model(&User{}).Count(&total)
    db.Limit(pageSize).Offset(offset).Find(&users)

    c.JSON(200, gin.H{
        "data": users,
        "total": total,
        "page": page,
        "pageSize": pageSize,
    })
}
```

**修改难度**: ⭐ 极简单
