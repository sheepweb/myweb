# 数据库索引优化 - 生产环境安全执行指南

## ⚠️ 重要提示

**在生产环境执行前，请务必备份数据库！**

## 📋 执行步骤

### 步骤 1: 备份数据库（必须！）

```bash
# 进入项目目录
cd /path/to/your/project

# 备份数据库
cp cboard.db cboard.db.backup_$(date +%Y%m%d_%H%M%S)

# 验证备份
ls -lh cboard.db*
```

### 步骤 2: 测试连接

```bash
# 测试数据库是否正常
sqlite3 cboard.db "SELECT COUNT(*) FROM users;"
```

### 步骤 3: 执行索引优化

```bash
# 执行索引 SQL
sqlite3 cboard.db < docs/sql/performance_indexes_sqlite.sql

# 或者逐条执行（更安全）
sqlite3 cboard.db <<EOF
-- 用户表索引
CREATE INDEX IF NOT EXISTS idx_users_is_active ON users(is_active);
CREATE INDEX IF NOT EXISTS idx_users_created_at ON users(created_at);

-- 订单表索引
CREATE INDEX IF NOT EXISTS idx_orders_user_id ON orders(user_id);
CREATE INDEX IF NOT EXISTS idx_orders_status ON orders(status);

-- 订阅表索引
CREATE INDEX IF NOT EXISTS idx_subscriptions_user_id ON subscriptions(user_id);
CREATE INDEX IF NOT EXISTS idx_subscriptions_status ON subscriptions(status);
EOF
```

### 步骤 4: 验证索引

```bash
# 查看已创建的索引
sqlite3 cboard.db "SELECT name, tbl_name FROM sqlite_master WHERE type='index' AND name LIKE 'idx_%';"
```

### 步骤 5: 测试查询速度

```bash
# 测试查询（应该更快）
sqlite3 cboard.db "SELECT COUNT(*) FROM users WHERE is_active = 1;"
sqlite3 cboard.db "SELECT COUNT(*) FROM orders WHERE status = 'paid';"
```

## 🔄 如果出现问题，立即回滚

```bash
# 停止服务
systemctl stop cboard  # 或 pkill cboard

# 恢复备份
cp cboard.db.backup_XXXXXX cboard.db

# 重启服务
systemctl start cboard
```

## ✅ 执行时机建议

1. **低峰期执行**（凌晨 2-4 点）
2. **提前通知用户**（维护公告）
3. **准备回滚方案**
4. **监控执行过程**

## 📊 预期效果

- 索引创建时间：1-2 分钟
- 数据库大小增加：约 5-10%
- 查询速度提升：70-90%
- 对现有数据：无影响

## 🚨 注意事项

1. **索引不会修改数据**，只是添加查询加速结构
2. **使用 IF NOT EXISTS**，重复执行不会报错
3. **可以随时删除索引**，不影响数据
4. **建议在测试环境先执行**

## 🧪 测试环境验证（推荐）

如果有测试环境，先在测试环境执行：

```bash

scp user@prod:/path/cboard.db /path/test/cboard.db

# 2. 在测试环境执行索引

# 4. 确认无误后，再在生产环境执行
```
