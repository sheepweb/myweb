# IP 地理位置查询服务参考

## 当前已实现的服务

1. **MaxMind GeoLite2** (本地数据库)
   - 优点：离线查询，速度快，无 API 限制
   - 缺点：需要定期更新数据库文件
   - 状态：✅ 已实现

2. **Ping0.cc API**
   - 优点：免费，支持 IPv4/IPv6，中文友好
   - 缺点：可能有速率限制
   - 状态：✅ 已实现

3. **IPW.cn API** (仅 IPv6)
   - 优点：专门支持 IPv6，中文友好
   - 缺点：仅支持 IPv6
   - 状态：✅ 已实现

## 推荐的备选服务

### 免费服务（有速率限制）

1. **ip-api.com**
   - URL: `http://ip-api.com/json/{ip}?lang=zh-CN`
   - 免费额度：45 请求/分钟
   - 支持：IPv4/IPv6，中文响应
   - 返回字段：国家、地区、城市、ISP、ASN 等
   - 优点：稳定、准确、支持中文
   - 示例响应：
     ```json
     {
       "status": "success",
       "country": "中国",
       "regionName": "北京",
       "city": "北京",
       "isp": "中国电信"
     }
     ```

2. **ipapi.co**
   - URL: `https://ipapi.co/{ip}/json/`
   - 免费额度：1000 请求/天
   - 支持：IPv4/IPv6
   - 返回字段：国家、地区、城市、时区、货币等
   - 优点：免费额度较高
   - 缺点：默认英文，需要额外处理

3. **ipinfo.io**
   - URL: `https://ipinfo.io/{ip}/json`
   - 免费额度：50,000 请求/月
   - 支持：IPv4/IPv6
   - 返回字段：国家、地区、城市、邮编、时区、ISP 等
   - 优点：免费额度高，数据准确
   - 缺点：需要注册获取 token（免费）

4. **ipwhois.app**
   - URL: `https://ipwhois.app/json/{ip}`
   - 免费额度：10,000 请求/月
   - 支持：IPv4/IPv6
   - 返回字段：国家、地区、城市、ISP、ASN 等
   - 优点：免费额度较高

5. **ip.sb**
   - URL: `https://api.ip.sb/geoip/{ip}`
   - 免费额度：无明确限制（合理使用）
   - 支持：IPv4/IPv6
   - 返回字段：国家、地区、城市、ISP 等
   - 优点：简单快速，中文友好
   - 缺点：文档较少

6. **geojs.io**
   - URL: `https://get.geojs.io/v1/ip/geo/{ip}.json`
   - 免费额度：无明确限制
   - 支持：IPv4/IPv6
   - 返回字段：国家、地区、城市、时区等
   - 优点：完全免费，无注册要求

### 国内服务（中文友好）

7. **ipip.net**
   - URL: `https://freeapi.ipip.net/{ip}`
   - 免费额度：有限制
   - 支持：IPv4/IPv6
   - 返回：数组格式 `["国家", "省份", "城市", "运营商"]`
   - 优点：国内 IP 数据准确
   - 缺点：需要解析数组格式

8. **ip138.com**
   - URL: `https://m.ip138.com/ip.asp?ip={ip}`
   - 免费额度：有限制
   - 支持：IPv4
   - 返回：HTML 格式，需要解析
   - 优点：国内 IP 数据准确
   - 缺点：需要解析 HTML

### 付费服务（高精度）

9. **MaxMind GeoIP2 Web Service**
   - 优点：高精度，商业级
   - 缺点：需要付费
   - 适用：商业项目

10. **IP2Location**
    - 优点：高精度，多种数据库
    - 缺点：需要付费
    - 适用：商业项目

## 推荐集成顺序

1. **ip-api.com** - 稳定、准确、中文支持好
2. **ip.sb** - 简单快速，中文友好
3. **ipinfo.io** - 免费额度高（需要注册）
4. **ipip.net** - 国内 IP 准确度高

## 注意事项

1. **速率限制**：所有免费服务都有速率限制，需要合理使用
2. **错误处理**：需要处理 API 失败、超时等情况
3. **缓存机制**：建议缓存查询结果，避免重复请求
4. **隐私合规**：确保符合 GDPR 等隐私法规
5. **备用方案**：实现多个备选服务，确保高可用性
