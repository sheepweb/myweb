# 采集地址配置说明

## 概述

节点「采集」是指从指定的订阅地址（URL）自动拉取节点列表并导入到系统。配置采集地址后，可在节点管理中使用「立即采集」或定时任务更新节点。

## 配置位置

- **管理员后台** → **配置更新**（或 **节点管理** 相关菜单下的配置更新/节点采集设置）

在「配置更新」页面中，有一项 **节点源 URL**（或「采集地址」）配置，用于填写要采集的订阅链接。

## 配置方法

1. 登录管理员后台，进入 **配置更新** 页面。
2. 在 **节点源 URL** 列表中，每行填写一个可访问的订阅地址，例如：
   - `https://example.com/subscribe?token=xxx`
   - 支持返回 Base64 编码节点列表或明文节点链接的订阅地址。
3. 可添加多个 URL，系统会按顺序采集并合并节点；同一批采集的节点会按 URL 顺序分配排序索引（OrderIndex）。
4. 保存配置后，在节点管理或配置更新页面点击 **立即采集** 即可从这些地址拉取并导入节点。

## 采集逻辑说明

- 系统会请求每个 URL，对返回内容进行 Base64 解码（如需要）并解析节点链接（如 vmess、vless、trojan、ss、ssr、hysteria 等）。
- 解析出的节点会与现有节点按「类型:服务器:端口」去重：已存在则更新，不存在则新建。
- **定时采集**执行时，会先清理所有「采集」来源的节点，再导入本次采集结果；**手动添加**的节点不会被清理。

## 注意事项

1. 订阅地址必须可被服务器访问（若为外网地址，需保证服务器网络可达）。
2. 订阅返回格式需为系统支持的协议链接或 Base64 编码的链接列表。
3. 采集频率不宜过高，避免对订阅源造成压力；定时任务建议 30 分钟～数小时一次。
4. 若部分 URL 采集失败，可查看后台日志或「配置更新」页的采集结果提示，排查地址或网络问题。
