# 注册设置说明

## 概述

注册设置用于控制用户注册流程、密码要求、新用户默认订阅等。合理配置注册设置可以提高系统安全性和用户体验。

## 配置位置

- **管理员后台** → **系统设置** → **注册设置**

## 一、基础注册设置

### 1. 开放注册

- **开启**：允许用户自行注册账号
- **关闭**：禁止新用户注册，只能由管理员创建账号

**使用场景：**
- **开启**：适合公开服务，允许用户自由注册
- **关闭**：适合内测、邀请制或完全由管理员管理的场景

### 2. 邮箱验证

- **开启**：新用户注册后必须验证邮箱才能激活账号
- **关闭**：注册后直接激活，无需验证邮箱

**建议：**
- 生产环境建议开启邮箱验证，提高账号安全性
- 确保已正确配置邮件服务器（见 [邮件服务器配置说明](./邮件服务器配置说明.md)）

### 3. 最小密码长度

- **默认值**：8 位
- **建议值**：8-16 位
- **说明**：用户注册时密码必须达到此长度

**安全建议：**
- 建议设置为 8 位或以上
- 密码长度越长越安全，但过长可能影响用户体验

### 4. 邀请码注册

- **开启**：用户注册时必须输入有效的邀请码
- **关闭**：用户注册时不需要邀请码

**使用场景：**
- **开启**：适合邀请制服务，控制注册用户数量和质量
- **关闭**：适合公开注册的服务

**配合使用：**
- 开启后，管理员需要在 **邀请管理** 中生成邀请码
- 用户注册时输入邀请码，系统会验证邀请码是否有效

## 二、新用户默认订阅设置

### 1. 默认设备数

- **默认值**：3
- **说明**：新注册用户默认订阅允许的最大设备数量

**配置建议：**
- 根据业务需求设置，一般 3-5 个设备足够
- 设备数过多可能影响服务质量
- 管理员可在用户注册后手动调整设备限制

**影响：**
- 用户注册后，系统会自动创建订阅，设备限制为此值
- 用户可以在订阅管理页面查看设备限制
- 超过设备限制时，用户无法添加新设备（见 [设备管理功能说明](../功能/设备管理功能说明.md)）

### 2. 默认订阅时长（月）

- **默认值**：1 个月
- **说明**：新注册用户默认订阅的有效期（从注册时间开始计算）

**配置建议：**
- 根据业务模式设置：
  - **试用模式**：设置为 0.5-1 个月，给用户试用期
  - **付费模式**：设置为 0（不自动创建订阅），用户需购买套餐
  - **免费模式**：设置为较长时长（如 3-12 个月）

**影响：**
- 用户注册后，系统会自动创建订阅，到期时间 = 注册时间 + 默认订阅时长
- 订阅到期后，用户无法使用服务，需要续费或购买新套餐

## 三、配置示例

### 示例 1：公开注册 + 邮箱验证

```
开放注册：✅ 开启
邮箱验证：✅ 开启
最小密码长度：8
邀请码注册：❌ 关闭
默认设备数：3
默认订阅时长：1 个月
```

**适用场景：** 公开服务，允许用户自由注册，需要邮箱验证提高安全性。

### 示例 2：邀请制 + 无需邮箱验证

```
开放注册：✅ 开启
邮箱验证：❌ 关闭
最小密码长度：8
邀请码注册：✅ 开启
默认设备数：5
默认订阅时长：3 个月
```

**适用场景：** 邀请制服务，通过邀请码控制注册，无需邮箱验证（快速注册）。

### 示例 3：管理员创建 + 试用期

```
开放注册：❌ 关闭
邮箱验证：✅ 开启
最小密码长度：8
邀请码注册：❌ 关闭
默认设备数：3
默认订阅时长：0.5 个月（15 天）
```

**适用场景：** 完全由管理员管理用户，新用户有 15 天试用期。

### 示例 4：付费模式（不自动创建订阅）

```
开放注册：✅ 开启
邮箱验证：✅ 开启
最小密码长度：8
邀请码注册：❌ 关闭
默认设备数：0（或 3，但订阅时长为 0）
默认订阅时长：0 个月
```

**适用场景：** 用户注册后必须购买套餐才能使用服务。

## 四、注册流程说明

### 流程 1：开启邮箱验证

```
1. 用户填写注册信息（邮箱、用户名、密码）
 ↓
2. 系统验证信息（密码长度、邮箱格式等）
 ↓
3. 系统创建账号（状态：待激活）
 ↓
4. 系统发送验证邮件到用户邮箱
 ↓
5. 用户点击邮件中的验证链接
 ↓
6. 系统激活账号，创建默认订阅
 ↓
7. 用户可以使用服务
```

### 流程 2：关闭邮箱验证

```
1. 用户填写注册信息（邮箱、用户名、密码）
 ↓
2. 系统验证信息（密码长度、邮箱格式等）
 ↓
3. 系统创建账号（状态：已激活）
 ↓
4. 系统自动创建默认订阅
 ↓
5. 用户可以直接使用服务
```

### 流程 3：开启邀请码注册

```
1. 用户填写注册信息 + 邀请码
 ↓
2. 系统验证邀请码是否有效
 ↓
3. 如果邀请码无效，注册失败
 ↓
4. 如果邀请码有效，继续注册流程
 ↓
5. （后续流程同流程 1 或 2）
```

## 五、注意事项

1. **邮箱验证依赖邮件服务器**：开启邮箱验证前，确保邮件服务器配置正确
2. **邀请码管理**：开启邀请码注册后，需要定期生成和管理邀请码
3. **默认订阅时长**：设置为 0 时，用户注册后没有可用订阅，需要购买套餐
4. **设备限制**：默认设备数会影响用户体验，建议根据实际需求设置
5. **密码长度**：过短不安全，过长影响体验，建议 8-16 位

## 六、常见问题

### Q1: 用户注册后收不到验证邮件？

**检查项：**
- 邮件服务器配置是否正确（见 [邮件服务器配置说明](./邮件服务器配置说明.md)）
- 用户邮箱是否正确
- 邮件是否被垃圾邮件拦截
- 系统日志中是否有发送失败的记录

### Q2: 如何批量创建用户？

**方法：**
- 管理员可以在 **用户管理** → **用户列表** 中手动创建用户
- 或使用脚本批量创建（需要开发支持）

### Q3: 邀请码在哪里生成？

**位置：**
- **管理员后台** → **邀请管理**（或类似菜单）
- 可以生成单个或批量生成邀请码
- 可以设置邀请码有效期、使用次数等

### Q4: 如何修改已注册用户的默认订阅？

**方法：**
- 在 **订阅管理** → **订阅列表** 中找到用户的订阅
- 点击「编辑订阅」，修改设备限制或到期时间
- 或使用「快捷添加时间」功能快速续费
