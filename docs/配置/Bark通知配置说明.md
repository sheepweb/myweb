# Bark 通知配置说明

## 概述

Bark 是一款 iOS 推送通知服务，系统支持通过 Bark 向管理员发送推送通知。配置后，管理员可在 iPhone/iPad 上实时接收系统通知，包括订单支付、用户注册、订阅到期等重要事件。

## 配置位置

- **管理员后台** → **系统设置** → **通知设置** → **管理员通知** → **Bark 通知**

## 一、安装 Bark App

### 1. 下载 Bark

1. 在 iPhone/iPad 上打开 **App Store**。
2. 搜索 **"Bark"** 或访问 [Bark 官网](https://github.com/Finb/Bark)。
3. 下载并安装 Bark App。

### 2. 打开 Bark 并获取 Device Key

1. 首次打开 Bark App，会自动生成一个 **Device Key**。
2. Device Key 显示在 App 首页，格式类似：
   ```
   AbCdEfGhIjKlMnOpQrStUvWxYz
   ```
3. **复制并保存** Device Key，后续配置需要用到。

### 3. 选择服务器（可选）

Bark 支持自建服务器或使用官方服务器：

- **官方服务器**（默认）：`https://api.day.app`
- **自建服务器**：如果您有自己的 Bark 服务器，使用自建地址

## 二、系统后台配置

### 1. 启用 Bark 通知

1. 登录管理员后台，进入 **系统设置** → **通知设置** → **管理员通知**。
2. 确保 **启用管理员通知** 开关已打开。
3. 打开 **Bark 通知** 开关。

### 2. 填写配置信息

- **服务器地址**：
  - 使用官方服务器：填写 `https://api.day.app`（默认值）
  - 使用自建服务器：填写您的 Bark 服务器地址（如 `https://your-bark-server.com`）
- **Device Key**：粘贴从 Bark App 中复制的 Device Key。

### 3. 测试通知

1. 点击 **测试 Bark 通知** 按钮。
2. 系统会发送一条测试推送到您的 iPhone/iPad。
3. 如果收到推送，说明配置成功；如果失败，检查服务器地址和 Device Key 是否正确。

### 4. 配置通知事件开关

在 **通知事件开关** 区域，可开启/关闭以下事件的通知：

- **订单支付成功**：用户完成支付时推送
- **新用户注册**：有新用户注册时推送
- **重置密码**：用户重置密码时推送
- **发送订阅**：向用户发送订阅邮件时推送
- **重置订阅**：用户或管理员重置订阅时推送
- **订阅到期**：订阅到期时推送
- **管理员创建用户**：管理员创建新用户时推送
- **订阅创建**：创建新订阅时推送

### 5. 保存配置

点击 **保存管理员通知设置** 按钮保存配置。

## 三、通知消息格式

系统发送的 Bark 推送包含：

- **标题**：事件类型（如「订单支付成功」「新用户注册」）
- **内容**：详细信息（订单号、用户邮箱、金额等）
- **时间戳**：事件发生时间

示例推送：
```
标题: 📦 订单支付成功

内容:
订单号: ORD20260210123456
用户: user@example.com
金额: ¥99.00
支付方式: 支付宝
时间: 2026-02-10 14:30:00
```

## 四、常见问题

### Q1: 收不到测试推送？

**检查项：**
1. Device Key 是否正确（从 Bark App 首页复制）
2. 服务器地址是否正确（官方：`https://api.day.app`）
3. iPhone/iPad 是否已连接网络
4. Bark App 是否已打开并允许通知权限

**解决方法：**
- 重新从 Bark App 复制 Device Key
- 检查 iPhone 设置 → 通知 → Bark，确保通知权限已开启
- 确认服务器地址格式正确（包含 `https://`）

### Q2: 只收到部分通知？

**检查项：**
- 在 **通知事件开关** 中确认对应事件已开启
- 检查系统日志，查看是否有发送失败的记录

### Q3: 如何更换接收通知的设备？

**方法：**
1. 在新设备上安装 Bark App，获取新的 Device Key
2. 在系统后台更新 Device Key
3. 保存配置并测试

### Q4: 可以使用多个设备接收通知吗？

**方法：**
- 每个设备有独立的 Device Key
- 如需多设备接收，需要配置多个 Bark 通知（如果系统支持），或使用其他通知方式（如 Telegram、邮件）作为补充

### Q5: 自建 Bark 服务器如何配置？

**步骤：**
1. 部署 Bark 服务器（参考 [Bark 官方文档](https://github.com/Finb/Bark)）
2. 在系统后台的 **服务器地址** 中填写您的服务器地址
3. 确保服务器地址可从外网访问（HTTPS）
4. 测试推送功能

## 五、安全建议

1. **保护 Device Key**：Device Key 是推送的唯一标识，不要泄露
2. **使用 HTTPS**：确保服务器地址使用 HTTPS 协议
3. **定期检查**：定期测试通知功能，确保正常工作
4. **官方服务器**：如无特殊需求，建议使用官方服务器 `https://api.day.app`

## 六、Bark App 使用技巧

### 1. 通知分组

在 Bark App 中可以设置通知分组，方便管理不同类型的通知。

### 2. 通知历史

Bark App 会保存推送历史，可以查看历史通知记录。

### 3. 自定义声音

可以在 Bark App 中为不同类型的通知设置不同的提示音。
