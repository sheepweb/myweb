# 支付宝配置说明

## 概述

系统支持支付宝电脑网站支付与手机网站支付。需在支付宝开放平台创建应用并获取 AppID、应用私钥，并配置支付宝公钥与回调地址。本文说明后台配置与支付宝开放平台侧的设置。

## 一、支付宝开放平台配置

### 1. 创建应用与获取 AppID

1. 登录 [支付宝开放平台](https://open.alipay.com/)。
2. 进入「开发者中心」→「网页/移动应用」→ 创建应用（如「网页应用」或「移动应用」），提交审核通过后生效。
3. 在应用详情中查看并复制 **AppID**。

### 2. 配置密钥（RSA2）

- **应用私钥**：由您本地生成并保存，用于请求支付宝时签名。
  - 开放平台文档推荐使用 RSA2(SHA256) 密钥长度 2048 位。
  - 可使用支付宝提供的「开发助手」或 OpenSSL 生成：
    ```bash
    openssl genrsa -out app_private_key.pem 2048
    ```
- **应用公钥**：由应用私钥导出，需上传到开放平台。
  - 在应用详情 → 「接口加签方式」→ 设置「公钥」：粘贴您生成的**应用公钥**内容（PEM 格式，含 BEGIN/END 行）。
- **支付宝公钥**：上传应用公钥后，开放平台会给出「支付宝公钥」，用于验证支付宝返回与回调的签名。请复制保存，并在系统后台「支付宝公钥」中填写。

### 3. 开放能力与签约

- 为应用添加「电脑网站支付」「手机网站支付」等能力并签约。
- 若使用沙箱环境，在「沙箱环境」中查看沙箱 AppID、沙箱支付宝公钥，并配置沙箱应用公钥。

### 4. 回调与授权（若需要）

- 支付结果通过「异步通知」回调到您配置的**异步通知地址**；「同步返回」仅用于跳转，不可依赖其作为支付结果依据。
- 在开放平台或应用配置中，如需填写通知地址，请与系统后台配置的异步回调地址一致（见下文）。

## 二、系统后台配置

### 1. 进入支付配置

- 登录管理员后台 → **支付配置**（或 系统设置 → 支付方式）。
- 新增或编辑「支付宝」支付方式。

### 2. 必填项

- **AppID**：支付宝开放平台应用的 AppID（沙箱即沙箱 AppID）。
- **应用私钥（商户私钥）**：您生成的 RSA 私钥完整内容（PEM 格式，含 `-----BEGIN ... -----` 与 `-----END ... -----`）。
- **支付宝公钥**：开放平台提供的「支付宝公钥」完整内容（PEM 格式），用于验证回调与同步返回的签名。

### 3. 选填项

- **异步通知地址（NotifyURL）**：建议显式配置为  
  `https://您的域名/api/v1/payment/notify/alipay`  
  若留空，部分接口会使用支付宝后台配置的地址或根据当前请求域名生成，生产环境建议固定填写。
- **同步返回地址（ReturnURL）**：支付完成后浏览器跳转地址，如  
  `https://您的域名/api/v1/payment/success` 或您的前台订单结果页。

### 4. 沙箱与正式环境

- **沙箱**：AppID、应用私钥、支付宝公钥均使用沙箱环境下的值；系统会根据配置或网关地址判断是否沙箱（如含 `alipaydev.com` 视为沙箱）。
- **正式**：使用正式应用 AppID、正式应用私钥与正式支付宝公钥，并确保签约了相应产品。

### 5. 保存与验证

- 保存后，可创建一笔小额订单选择支付宝支付，完成支付后检查订单是否自动变为「已支付」。
- 若回调验证失败，请检查：支付宝公钥是否为当前应用/环境对应的公钥、私钥与 AppID 是否匹配、异步通知地址是否可被支付宝访问（HTTPS、无鉴权）。

## 三、常见问题

- **提示「应用私钥未配置」或「私钥格式错误」**  
  请确认在后台填写的是完整 PEM（含 BEGIN/END），且为 PKCS#1 或 PKCS#8 格式；若为 PKCS#8，系统一般可识别。

- **支付成功但订单未更新**  
  多为异步回调未到达或验签失败。请确认异步通知地址可从外网访问、使用 HTTPS，并检查支付宝公钥是否正确、是否与当前环境（沙箱/正式）一致；查看系统日志中的支付宝回调与验签错误信息。

- **沙箱与正式环境区别**  
  沙箱使用沙箱 AppID、沙箱密钥和沙箱支付宝公钥，并仅在沙箱环境中完成支付；正式环境需使用正式应用与签约。
