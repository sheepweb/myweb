# 主题设置说明

## 概述

主题设置用于配置网站的外观主题，包括默认主题、用户自定义主题权限、可用主题列表等。合理配置主题设置可以提升用户体验和品牌形象。

## 配置位置

- **管理员后台** → **系统设置** → **主题设置**

## 一、默认主题

### 1. 默认主题选择

系统提供以下主题选项：

- **浅色主题**：适合日间使用，明亮清晰
- **深色主题**：适合夜间使用，护眼舒适
- **蓝色主题**：专业商务风格
- **绿色主题**：清新自然风格
- **紫色主题**：优雅神秘风格
- **橙色主题**：活力热情风格
- **红色主题**：醒目警示风格
- **青色主题**：清新科技风格
- **Luck 主题**：金色幸运风格
- **Aurora 主题**：极光梦幻风格
- **跟随系统**：根据用户设备的系统设置自动切换（浅色/深色）

**说明：**
- 默认主题是新用户首次访问时使用的主题
- 用户可以在个人设置中切换主题（如果允许）

## 二、用户自定义主题权限

### 1. 允许用户自定义

- **开启**：允许用户在个人设置中切换主题
- **关闭**：用户无法切换主题，只能使用默认主题

**使用场景：**
- **开启**：适合需要个性化体验的场景，用户可以选择喜欢的主题
- **关闭**：适合需要统一品牌形象的场景，所有用户使用相同主题

## 三、可用主题列表

### 1. 可用主题配置

- **说明**：选择哪些主题对用户可见和可选
- **格式**：多选框，可以勾选多个主题

**配置方法：**
1. 勾选希望用户可用的主题
2. 未勾选的主题不会显示在用户的主题选择列表中
3. 默认主题必须包含在可用主题列表中

**使用场景：**
- **全部可用**：用户可以选择任意主题
- **部分可用**：只允许用户选择部分主题（如只允许浅色和深色）
- **单一主题**：只允许一个主题，实现统一外观

## 四、主题配置示例

### 示例 1：允许用户自由选择

```
默认主题：跟随系统
允许用户自定义：✅ 开启
可用主题：
  ✅ 浅色主题
  ✅ 深色主题
  ✅ 蓝色主题
  ✅ 绿色主题
  ✅ 紫色主题
  ✅ 橙色主题
  ✅ 红色主题
  ✅ 青色主题
  ✅ Luck 主题
  ✅ Aurora 主题
  ✅ 跟随系统
```

### 示例 2：限制为浅色和深色

```
默认主题：浅色主题
允许用户自定义：✅ 开启
可用主题：
  ✅ 浅色主题
  ✅ 深色主题
  ✅ 跟随系统
  ❌ 其他主题（不勾选）
```

### 示例 3：统一主题（不允许切换）

```
默认主题：蓝色主题
允许用户自定义：❌ 关闭
可用主题：
  ✅ 蓝色主题（必须勾选默认主题）
  ❌ 其他主题（不勾选）
```

## 五、主题切换说明

### 用户端主题切换

如果「允许用户自定义」已开启：

1. 用户登录后，进入个人设置或用户中心
2. 找到「主题设置」或「外观设置」
3. 从可用主题列表中选择喜欢的主题
4. 主题立即生效，无需刷新页面

### 主题持久化

- 用户选择的主题会保存到本地存储或用户设置中
- 下次登录时自动应用上次选择的主题
- 如果用户未选择，使用系统默认主题

## 六、注意事项

1. **默认主题必须可用**：默认主题必须包含在可用主题列表中
2. **用户体验**：建议至少提供浅色和深色两种主题，满足不同用户需求
3. **品牌一致性**：如果需要统一品牌形象，可以限制可用主题或关闭用户自定义
4. **跟随系统**：建议包含「跟随系统」选项，提供更好的用户体验

## 七、常见问题

### Q1: 用户无法切换主题？

**检查项：**
- 「允许用户自定义」是否已开启
- 用户选择的主题是否在可用主题列表中
- 清除浏览器缓存后重试

### Q2: 如何强制所有用户使用同一主题？

**方法：**
1. 设置默认主题为目标主题
2. 关闭「允许用户自定义」
3. 在可用主题列表中只勾选默认主题

### Q3: 主题显示异常？

**检查项：**
- 浏览器是否支持 CSS 变量（现代浏览器都支持）
- 清除浏览器缓存
- 检查浏览器控制台是否有错误信息
