# 公告管理说明

## 概述

公告管理用于在用户登录时显示重要通知、系统维护信息、活动公告等。公告以弹窗形式显示，确保用户能够及时看到重要信息。

## 配置位置

- **管理员后台** → **系统设置** → **公告管理**

## 一、启用公告

### 1. 启用公告开关

- **开启**：用户登录时会看到公告弹窗
- **关闭**：不显示公告弹窗

**使用场景：**
- **开启**：有重要通知需要告知用户时
- **关闭**：无重要通知时，避免打扰用户

## 二、公告内容

### 1. 公告内容编辑

- **格式**：支持 HTML 格式，可以添加链接、图片、样式等
- **长度**：建议不超过 500 字，避免弹窗过大
- **内容**：可以包含文字、链接、图片等

**HTML 示例：**
```html
<h3>系统维护通知</h3>
<p>系统将于 <strong>2026-02-15 02:00-04:00</strong> 进行维护，期间可能无法访问，请提前做好准备。</p>
<p>如有问题，请联系客服：<a href="mailto:support@example.com">support@example.com</a></p>
```

**纯文本示例：**
```
重要通知：系统将于 2026-02-15 进行升级维护，预计耗时 2 小时，请提前做好准备。
```

## 三、公告显示说明

### 1. 显示时机

- **用户登录时**：用户成功登录后，公告弹窗自动显示
- **每次登录**：每次登录都会显示（如果公告已启用）
- **关闭后**：用户关闭弹窗后，本次会话不再显示

### 2. 弹窗行为

- **模态弹窗**：用户必须关闭弹窗才能继续操作
- **可关闭**：用户可以点击关闭按钮或遮罩层关闭
- **不强制**：关闭后不会再次显示（除非重新登录）

## 五、HTML 使用技巧

### 1. 基本 HTML 标签

- **标题**：`<h1>`、`<h2>`、`<h3>` 等
- **段落**：`<p>`
- **强调**：`<strong>`（粗体）、`<em>`（斜体）
- **列表**：`<ul>`、`<ol>`、`<li>`
- **链接**：`<a href="url">文本</a>`
- **换行**：`<br>`

### 2. 样式建议

- **简洁明了**：避免过于复杂的样式
- **易读性**：确保文字清晰易读
- **响应式**：考虑移动端显示效果

### 3. 安全注意事项

- **避免恶意代码**：系统会过滤危险的 HTML 标签和脚本
- **链接安全**：确保链接指向可信的地址
- **内容审核**：发布前检查内容是否正确

## 六、注意事项

1. **及时关闭**：公告发布后，记得在不需要时关闭，避免长期显示
2. **内容更新**：定期更新公告内容，保持信息时效性
3. **用户体验**：避免过于频繁的公告，影响用户体验
4. **重要信息**：只用于重要通知，避免滥用

## 七、常见问题

### Q1: 公告不显示？

**检查项：**
- 「启用公告」是否已开启
- 公告内容是否为空
- 用户是否已登录
- 浏览器是否阻止了弹窗

### Q2: HTML 格式不正确？

**检查项：**
- HTML 标签是否闭合
- 是否有语法错误
- 特殊字符是否转义

**解决方法：**
- 使用简单的 HTML 标签
- 避免复杂的嵌套结构
- 测试后再发布

### Q3: 如何让公告只显示一次？

**说明：**
- 系统默认每次登录都会显示公告
- 如果需要只显示一次，可以在公告内容中说明「本次更新后不再显示」
- 或者使用其他通知方式（如站内消息）

### Q4: 公告可以包含图片吗？

**方法：**
- 可以，使用 `<img>` 标签
- 图片 URL 必须是可访问的（建议使用 HTTPS）
- 注意图片大小，避免影响加载速度

**示例：**
```html
<img src="https://example.com/announcement.png" alt="公告图片" style="max-width: 100%;">
```
