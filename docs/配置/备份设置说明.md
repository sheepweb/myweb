# 备份设置说明

## 概述

备份设置用于配置数据库自动备份到 Gitee 或 GitHub，确保数据安全。系统支持自动备份和手动备份，备份文件会自动上传到远程仓库。

## 配置位置

- **管理员后台** → **系统设置** → **备份设置**

## 一、备份目标选择

### 1. 备份目标平台

系统支持两种备份目标：

- **Gitee**：码云（适合国内服务器）
- **GitHub**：GitHub（适合海外服务器或需要国际访问的场景）

**选择建议：**
- **国内服务器**：建议使用 Gitee，访问速度快
- **海外服务器**：可以使用 GitHub 或 Gitee
- **双备份**：可以同时配置两个平台，提高数据安全性

## 二、Gitee 配置

### 1. 启用 Gitee 备份

- **开启**：备份文件会自动上传到 Gitee 仓库
- **关闭**：不使用 Gitee 备份

### 2. Gitee Access Token

- **获取方法**：
  1. 登录 [Gitee](https://gitee.com/)
  2. 进入 **设置** → **安全设置** → **私人令牌**
  3. 点击 **生成新令牌**
  4. 勾选 **projects** 权限（需要仓库读写权限）
  5. 生成后立即复制 Token（只显示一次）

- **格式**：`gitee_xxxxxxxxxxxxxxxxxxxxx`

### 3. 仓库所有者

- **说明**：Gitee 用户名或组织名
- **示例**：`your-username`、`your-org`

### 4. 仓库名称

- **说明**：备份仓库名称
- **示例**：`backup`、`cboard-backup`
- **格式**：备份文件会保存为 `YYYY-MM-DD/backup.zip` 格式

### 5. 测试连接

- **功能**：测试 Gitee Token 和仓库配置是否正确
- **方法**：点击「测试连接」按钮
- **说明**：测试成功后才能正常备份

## 三、GitHub 配置

### 1. 启用 GitHub 备份

- **开启**：备份文件会自动上传到 GitHub 仓库
- **关闭**：不使用 GitHub 备份

### 2. GitHub Personal Access Token

- **获取方法**：
  1. 登录 [GitHub](https://github.com/)
  2. 进入 **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
  3. 点击 **Generate new token (classic)**
  4. 勾选 **repo** 权限（需要仓库读写权限）
  5. 生成后立即复制 Token（只显示一次）

- **格式**：`ghp_xxxxxxxxxxxxxxxxxxxxx`

### 3. 仓库所有者

- **说明**：GitHub 用户名或组织名
- **示例**：`your-username`、`your-org`

### 4. 仓库名称

- **说明**：备份仓库名称
- **示例**：`backup`、`cboard-backup`
- **格式**：备份文件会保存为 `YYYY-MM-DD/backup.zip` 格式

### 5. 测试连接

- **功能**：测试 GitHub Token 和仓库配置是否正确
- **方法**：点击「测试连接」按钮
- **说明**：测试成功后才能正常备份

## 四、自动备份设置

### 1. 启用自动备份

- **开启**：系统会按设定间隔自动备份
- **关闭**：只进行手动备份

### 2. 备份间隔（小时）

- **默认值**：24 小时
- **说明**：每隔多长时间自动备份一次

**建议值：**
- **高频备份**：12 小时（适合数据变化频繁的场景）
- **标准备份**：24 小时（推荐，平衡备份频率和存储空间）
- **低频备份**：48 小时或更长（适合数据变化不频繁的场景）

**注意事项：**
- 备份频率过高会增加仓库存储空间
- 备份频率过低可能丢失较多数据
- 建议根据数据重要性和变化频率调整

## 五、手动备份

### 1. 立即备份

- **功能**：立即创建备份并上传（如果已启用远程备份）
- **方法**：点击「立即备份」按钮
- **说明**：
  - 备份文件会立即创建
  - 如果启用了 Gitee 或 GitHub 备份，会自动上传
  - 上传过程可能需要几分钟，可以查看上传进度

### 2. 备份文件格式

- **文件名**：`YYYY-MM-DD/backup.zip`
- **内容**：数据库文件（SQLite 或 MySQL/PostgreSQL 导出文件）
- **大小**：根据数据库大小，一般几 MB 到几十 MB

## 六、备份流程说明

### 1. 自动备份流程

```
1. 系统按设定间隔触发备份任务
 ↓
2. 创建数据库备份文件
 ↓
3. 如果启用了 Gitee 备份：
   - 上传到 Gitee 仓库
   - 保存为 YYYY-MM-DD/backup.zip
 ↓
4. 如果启用了 GitHub 备份：
   - 上传到 GitHub 仓库
   - 保存为 YYYY-MM-DD/backup.zip
 ↓
5. 备份完成，记录备份时间
```

### 2. 手动备份流程

```
1. 管理员点击「立即备份」按钮
 ↓
2. 系统立即创建备份文件
 ↓
3. 如果启用了远程备份，开始上传
 ↓
4. 显示上传进度（如果支持）
 ↓
5. 上传完成，显示备份信息
```

## 八、注意事项

1. **Token 安全**：Token 等同于密码，不要泄露或提交到代码仓库
2. **仓库权限**：确保 Token 有仓库读写权限（Gitee 需要 projects，GitHub 需要 repo）
3. **仓库创建**：备份前确保仓库已创建（系统不会自动创建仓库）
4. **网络连接**：确保服务器可以访问 Gitee/GitHub API
5. **存储空间**：定期清理旧备份，避免仓库存储空间过大

## 九、常见问题

### Q1: 备份上传失败？

**检查项：**
- Token 是否正确
- Token 是否过期
- 仓库所有者名称是否正确
- 仓库名称是否正确
- 服务器网络是否正常
- Token 是否有仓库读写权限

**解决方法：**
- 重新生成 Token
- 检查仓库信息
- 测试连接功能
- 查看系统日志中的错误信息

### Q2: 如何恢复备份？

**方法：**
1. 从 Gitee/GitHub 仓库下载备份文件
2. 解压备份文件，获取数据库文件
3. 停止系统服务
4. 替换数据库文件（注意备份当前数据库）
5. 重启系统服务

**注意事项：**
- 恢复备份会覆盖当前数据，请谨慎操作
- 建议先备份当前数据库
- 确保备份文件版本正确

### Q3: 备份文件在哪里？

**位置：**
- **Gitee**：`https://gitee.com/your-username/backup/tree/main/YYYY-MM-DD`
- **GitHub**：`https://github.com/your-username/backup/tree/main/YYYY-MM-DD`
- **本地**：备份文件创建在服务器临时目录，上传后可能被清理

### Q4: 如何清理旧备份？

**方法：**
- **Gitee**：在仓库中手动删除旧备份文件夹
- **GitHub**：在仓库中手动删除旧备份文件夹
- **自动清理**：可以配置自动清理策略（如果系统支持）

**建议：**
- 保留最近 7-30 天的备份
- 定期清理更早的备份
- 重要时间点的备份可以长期保留
