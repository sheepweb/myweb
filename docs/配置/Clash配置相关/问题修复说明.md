# 🔧 Clash 配置 Emoji 显示问题修复

## 问题描述

客户导入 Clash 配置时，emoji 字符显示为 Unicode 转义序列：
```yaml
- name: "\U0001F1ED\U0001F1F0 香港节点01"  # ❌ 错误
- name: "\U0001F680 节点选择"              # ❌ 错误
```

应该显示为：
```yaml
- name: "🇭🇰 香港节点01"  # ✅ 正确
- name: "🚀 节点选择"      # ✅ 正确
```

## 问题原因

Go 语言的 `yaml.Marshal()` 函数默认会将非 ASCII 字符（包括 emoji）转义为 Unicode 序列（`\UXXXXXXXX` 格式）。

## 解决方案

### 1. 添加 Unicode 反转义函数

在 `internal/services/config_update/config_update.go` 文件末尾添加：

```go
// unescapeUnicode 将 Unicode 转义序列还原为实际字符
// 例如：将 \U0001F1ED\U0001F1F0 还原为 🇭🇰
func unescapeUnicode(s string) string {
	// 匹配 \U 开头的 8 位十六进制 Unicode 转义序列
	re := regexp.MustCompile(`\\U([0-9A-Fa-f]{8})`)
	
	result := re.ReplaceAllStringFunc(s, func(match string) string {
		// 提取十六进制数字部分
		hexStr := match[2:] // 跳过 \U
		
		// 转换为整数
		codePoint, err := strconv.ParseInt(hexStr, 16, 64)
		if err != nil {
			return match // 如果解析失败，保持原样
		}
		
		// 转换为 rune（Unicode 字符）
		return string(rune(codePoint))
	})
	
	return result
}
```

## 技术细节

### Unicode 转义格式

- **\U 格式**: 8 位十六进制，如 `\U0001F1ED` 表示 🇭
- **范围**: U+00000000 到 U+0010FFFF
- **Emoji**: 通常在 U+1F000 到 U+1F9FF 范围

### 常见 Emoji 对应关系

| Emoji | Unicode | 转义序列 |
|-------|---------|----------|
| 🇭🇰 | U+1F1ED U+1F1F0 | \U0001F1ED\U0001F1F0 |
| 🇺🇸 | U+1F1FA U+1F1F8 | \U0001F1FA\U0001F1F8 |
| 🇯🇵 | U+1F1EF U+1F1F5 | \U0001F1EF\U0001F1F5 |
| 🚀 | U+1F680 | \U0001F680 |
| ♻️ | U+267B U+FE0F | ♻️ (直接显示) |
| 🔰 | U+1F530 | \U0001F530 |
| 🔮 | U+1F52E | \U0001F52E |
