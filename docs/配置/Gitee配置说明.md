# Gitee 配置说明

## 概述

系统支持将数据库备份上传到 Gitee（码云）仓库。Gitee 是国内代码托管平台，访问速度快，适合国内服务器使用。

## 配置位置

- **管理员后台** → **系统设置** → **备份设置** → **Gitee 配置**

## 一、Gitee 侧准备

### 1. 创建仓库

1. 登录 [Gitee](https://gitee.com/)。
2. 点击右上角 **+** → **新建仓库**。
3. 填写仓库信息：
   - **仓库名称**：例如 `cboard-backup` 或 `backup`
   - **仓库介绍**：可选，如「CBoard 数据库备份」
   - **可见性**：建议选择 **私有**（保护数据安全）
   - **初始化仓库**：可以不勾选「使用 Readme 文件初始化」（系统会自动创建文件）
4. 点击 **创建**。

### 2. 创建 Access Token

1. 登录 Gitee，点击右上角头像 → **设置**。
2. 左侧菜单选择 **安全设置** → **私人令牌**。
3. 点击 **生成新令牌**。
4. 填写令牌信息：
   - **令牌描述**：例如「CBoard 备份」
   - **权限范围**：**必须勾选 `projects`**（项目权限，用于仓库读写）
5. 点击 **提交**。
6. **立即复制** Token（格式：`gitee_xxxxxxxxxxxxxxxxxxxxx`），只显示一次。

## 二、系统后台配置

### 1. 启用 Gitee 备份

1. 登录管理员后台，进入 **系统设置** → **备份设置**。
2. 在 **备份目标选择** 中选择 **Gitee**（或保持默认）。
3. 打开 **启用 Gitee 备份** 开关。

### 2. 填写配置信息

- **Access Token**：粘贴从 Gitee 获取的 Token。
- **仓库所有者**：Gitee 用户名或组织名（如 `your-username`）。
- **仓库名称**：备份仓库名称（如 `backup` 或 `cboard-backup`）。

### 3. 测试连接

1. 点击 **测试连接** 按钮。
2. 系统会验证 Token 和仓库配置。
3. 如果显示「连接成功」，说明配置正确；如果失败，检查 Token 和仓库信息。

### 4. 配置自动备份（可选）

- **启用自动备份**：打开开关。
- **备份间隔**：设置自动备份间隔（建议 12-24 小时）。

### 5. 保存配置

点击 **保存备份设置** 按钮保存配置。

## 三、备份文件说明

### 1. 文件格式

- **路径**：`YYYY-MM-DD/backup.zip`
- **内容**：数据库文件（SQLite 的 `.db` 文件或 MySQL/PostgreSQL 的导出文件）
- **大小**：根据数据库大小，一般几 MB 到几十 MB

### 2. 文件命名规则

- **日期格式**：`YYYY-MM-DD`（如 `2026-02-10`）
- **文件名**：`backup.zip`
- **完整路径**：`2026-02-10/backup.zip`

## 四、手动备份

### 1. 立即备份

1. 在备份设置页面，点击 **立即备份** 按钮。
2. 系统会立即创建备份文件。
3. 如果启用了 Gitee 备份，会自动上传到 Gitee 仓库。
4. 可以查看上传进度（如果支持）。

### 2. 查看备份

备份完成后，可以在 Gitee 仓库中查看：

1. 访问：`https://gitee.com/your-username/backup`
2. 找到对应日期的文件夹（如 `2026-02-10`）
3. 下载 `backup.zip` 文件

## 五、常见问题

### Q1: 测试连接失败？

**检查项：**
- Token 是否正确（格式：`gitee_` 开头）
- Token 是否过期
- Token 是否有 `projects` 权限
- 仓库所有者名称是否正确
- 仓库名称是否正确
- 仓库是否存在

**解决方法：**
- 重新生成 Token，确保勾选 `projects` 权限
- 检查仓库信息是否正确
- 确认仓库已创建

### Q2: 备份上传失败？

**检查项：**
- Token 是否正确
- 仓库权限是否正确
- 服务器网络是否正常
- 查看系统日志中的错误信息

**解决方法：**
- 重新测试连接
- 检查网络连接
- 查看系统日志

### Q3: 如何恢复备份？

**步骤：**
1. 从 Gitee 仓库下载备份文件（`backup.zip`）
2. 解压获取数据库文件
3. 停止系统服务
4. 备份当前数据库（以防万一）
5. 替换数据库文件
6. 重启系统服务

**注意事项：**
- 恢复备份会覆盖当前数据，请谨慎操作
- 建议先备份当前数据库
- 确保备份文件版本正确

### Q4: 如何清理旧备份？

**方法：**
1. 登录 Gitee，进入备份仓库
2. 找到旧备份文件夹（如 `2026-01-01`）
3. 点击删除按钮删除文件夹

**建议：**
- 保留最近 7-30 天的备份
- 定期清理更早的备份
- 重要时间点的备份可以长期保留

## 六、安全建议

1. **使用私有仓库**：建议将备份仓库设置为私有，保护数据安全
2. **保护 Token**：Token 等同于密码，不要泄露
3. **定期更新 Token**：建议定期更换 Token，提高安全性
4. **限制仓库访问**：只允许必要的账号访问备份仓库
