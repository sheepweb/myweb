# 安全设置与限制说明

## 概述

安全设置用于保护系统免受暴力破解、未授权访问等安全威胁。包括登录失败限制、账户锁定、IP 白名单、会话超时等。合理配置安全设置可以显著提高系统安全性。

## 配置位置

- **管理员后台** → **系统设置** → **安全设置**

## 一、登录失败限制

### 1. 登录失败限制（次数）

- **默认值**：5 次
- **说明**：同一 IP 地址在指定时间内连续登录失败达到此次数后，会被临时锁定

**工作原理：**
- 系统记录每个 IP 的登录失败次数
- 连续失败达到限制次数后，该 IP 会被锁定
- 锁定期间无法继续尝试登录

**建议值：**
- **宽松**：10 次（适合内部使用）
- **标准**：5 次（推荐，平衡安全与体验）
- **严格**：3 次（适合高安全要求场景）

### 2. 锁定时间（分钟）

- **默认值**：30 分钟
- **说明**：IP 被锁定后，需要等待此时间才能重新尝试登录

**工作原理：**
- 登录失败达到限制次数后，IP 被锁定
- 锁定期间，该 IP 的所有登录请求都会被拒绝
- 锁定时间到期后，失败次数清零，可以重新尝试

**建议值：**
- **短期锁定**：15-30 分钟（推荐，平衡安全与用户体验）
- **中期锁定**：60 分钟（适合安全要求较高的场景）
- **长期锁定**：120 分钟或更长（适合遭受攻击时临时启用）

### 3. 系统内置速率限制

除了上述配置的限制外，系统还内置了速率限制器：

- **速率限制**：15 分钟内连续失败 5 次，IP 会被锁定 15 分钟
- **自动重置**：锁定时间到期后自动解除
- **独立于配置**：此限制独立于「登录失败限制」配置，两者同时生效

## 二、会话超时

### 1. 会话超时（分钟）

- **默认值**：120 分钟（2 小时）
- **说明**：用户登录后，如果在此时间内没有任何操作，会话会自动过期，需要重新登录

**工作原理：**
- 用户每次操作都会刷新会话时间
- 如果超过设定时间没有操作，会话过期
- 会话过期后，用户需要重新登录

**建议值：**
- **短超时**：30-60 分钟（高安全要求）
- **标准超时**：120 分钟（推荐，平衡安全与体验）
- **长超时**：240 分钟或更长（适合内部使用）

## 三、IP 白名单

### 1. 启用 IP 白名单

- **开启**：只有白名单中的 IP 可以访问系统
- **关闭**：所有 IP 都可以访问（受其他安全设置限制）

**使用场景：**
- **开启**：适合内网使用或特定 IP 访问的场景
- **关闭**：适合公开服务

### 2. IP 白名单配置

- **格式**：每行一个 IP 地址或 IP 段
- **支持格式**：
  - 单个 IP：`192.168.1.100`
  - IP 段：`192.168.1.0/24`（CIDR 格式）
  - 多个 IP：每行一个

**示例：**
```
192.168.1.100
192.168.1.0/24
10.0.0.1
172.16.0.0/16
```

**注意事项：**
- 启用 IP 白名单后，**只有白名单中的 IP 可以访问**
- 管理员自己的 IP 必须添加到白名单，否则无法登录
- 建议先添加管理员 IP，再启用白名单

## 四、账户解锁方法

### 方法一：等待自动解锁

- **适用场景**：IP 被速率限制器锁定
- **等待时间**：15 分钟
- **说明**：锁定时间到期后自动解除，无需操作

### 方法二：使用解锁脚本

- **适用场景**：用户账户被锁定（多次登录失败）
- **命令**：
  ```bash
  cd /path/to/project
  go run scripts/unlock_user.go <用户名或邮箱>
  ```
- **说明**：
  - 支持使用用户名或邮箱解锁
  - 可以解锁管理员账户和普通用户账户
  - 解锁操作会清除所有登录失败记录，设置账户为激活状态

**示例：**
```bash
# 解锁管理员账户
go run scripts/unlock_user.go admin

# 解锁普通用户
go run scripts/unlock_user.go user@example.com
```

### 方法三：修改 IP（绕过 IP 锁定）

- **适用场景**：IP 被锁定，无法登录
- **方法**：
  - 更换网络（使用移动网络、VPN 等）
  - 等待锁定时间到期
  - 联系管理员将您的 IP 添加到白名单（如果启用了白名单）

### 方法四：重启服务器（清除内存中的限制记录）

- **适用场景**：紧急情况，需要立即解除所有 IP 锁定
- **方法**：重启服务器（会清除内存中的速率限制记录）
- **注意**：此方法会影响所有用户，谨慎使用

## 五、安全设置配置示例

### 示例 1：标准安全配置（推荐）

```
登录失败限制：5 次
锁定时间：30 分钟
会话超时：120 分钟
启用 IP 白名单：❌ 关闭
```

**适用场景：** 公开服务，平衡安全与用户体验。

### 示例 2：高安全配置

```
登录失败限制：3 次
锁定时间：60 分钟
会话超时：60 分钟
启用 IP 白名单：✅ 开启
IP 白名单：
  192.168.1.0/24
  10.0.0.1
```

**适用场景：** 内网使用或高安全要求场景。

### 示例 3：宽松配置

```
登录失败限制：10 次
锁定时间：15 分钟
会话超时：240 分钟
启用 IP 白名单：❌ 关闭
```

**适用场景：** 内部测试或低安全要求场景。

## 六、安全建议

1. **合理设置失败限制**：不要设置过低（影响用户体验）或过高（降低安全性）
2. **启用 IP 白名单**：如果可能，建议启用 IP 白名单，限制访问来源
3. **定期检查日志**：定期查看系统日志，发现异常登录尝试
4. **使用强密码**：建议用户使用强密码（见注册设置中的最小密码长度）
5. **启用 HTTPS**：确保系统使用 HTTPS，保护登录凭据传输安全
6. **定期更新**：定期更新系统和依赖，修复安全漏洞

## 七、常见问题

### Q1: 忘记管理员密码，账户被锁定怎么办？

**解决方法：**
1. 使用解锁脚本解锁账户：
   ```bash
   go run scripts/unlock_user.go admin
   ```
2. 使用管理员工具重置密码：
   ```bash
   go run scripts/admin_tool.go <新密码>
   ```
3. 重新登录

### Q2: IP 被锁定，无法登录怎么办？

**解决方法：**
1. 等待锁定时间到期（15-30 分钟）
2. 更换 IP 地址（使用 VPN 或移动网络）
3. 联系其他管理员将您的 IP 添加到白名单（如果启用了白名单）
4. 紧急情况：重启服务器（会清除内存中的限制记录）

### Q3: 启用 IP 白名单后无法登录？

**检查项：**
- 当前 IP 是否在白名单中
- IP 格式是否正确
- 是否包含 CIDR 格式的 IP 段

**解决方法：**
- 通过其他方式（如服务器 SSH）访问系统
- 将当前 IP 添加到白名单
- 或临时关闭 IP 白名单，添加 IP 后再开启

### Q4: 如何查看登录失败记录？

**方法：**
- 查看系统日志（`server.log` 或 `uploads/logs/app.log`）
- 查看安全日志（如果有安全日志功能）
- 查看用户登录历史（管理员后台 → 用户管理 → 登录历史）

### Q5: 如何防止暴力破解攻击？

**建议：**
1. 设置较低的登录失败限制（3-5 次）
2. 设置较长的锁定时间（30-60 分钟）
3. 启用 IP 白名单（如果可能）
4. 使用强密码策略
5. 启用 HTTPS
6. 定期检查日志，发现异常及时处理

## 八、相关脚本说明

### unlock_user.go

**功能：** 解锁用户账户

**用法：**
```bash
go run scripts/unlock_user.go <用户名或邮箱>
```

**说明：**
- 清除所有登录失败记录
- 设置账户为激活状态
- 设置账户为已验证状态
- 支持管理员和普通用户

### admin_tool.go

**功能：** 创建/重置管理员账户和密码

**用法：**
```bash
# 使用环境变量
export ADMIN_USERNAME="admin"
export ADMIN_EMAIL="admin@example.com"
export ADMIN_PASSWORD="YourPassword123!"
go run scripts/admin_tool.go

# 或直接传密码
go run scripts/admin_tool.go YourPassword123!
```
