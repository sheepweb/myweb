# Telegram 通知配置说明

## 概述

系统支持通过 Telegram Bot 向管理员发送通知，包括订单支付成功、新用户注册、订阅到期等重要事件。配置后，管理员可在 Telegram 客户端实时接收系统通知。

## 配置位置

- **管理员后台** → **系统设置** → **通知设置** → **管理员通知** → **Telegram 通知**

## 一、创建 Telegram Bot

### 1. 获取 Bot Token

1. 打开 Telegram，搜索并联系 **@BotFather**（Telegram 官方机器人）。
2. 发送命令 `/newbot` 创建新机器人。
3. 按提示输入机器人名称（如：`My CBoard Bot`）。
4. 输入机器人用户名（必须以 `bot` 结尾，如：`my_cboard_bot`）。
5. BotFather 会返回 **Bot Token**，格式类似：
   ```
   1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
   ```
6. **立即复制并保存** Bot Token，只显示一次。

### 2. 配置 Bot（可选）

- 发送 `/setdescription` 设置机器人描述
- 发送 `/setabouttext` 设置简介
- 发送 `/setuserpic` 设置头像

## 二、获取 Chat ID

### 方法一：使用 @userinfobot（推荐）

1. 在 Telegram 中搜索并联系 **@userinfobot**。
2. 发送任意消息（如 `/start`）。
3. @userinfobot 会返回您的 **Chat ID**（数字，如 `123456789`）。
4. 复制并保存 Chat ID。

### 方法二：使用 @getidsbot

1. 搜索并联系 **@getidsbot**。
2. 发送任意消息。
3. 获取返回的 Chat ID。

### 方法三：通过 API 获取

1. 先向您的 Bot 发送一条消息（任意内容）。
2. 访问：`https://api.telegram.org/bot<您的BotToken>/getUpdates`
3. 在返回的 JSON 中找到 `"chat":{"id":123456789}`，其中的数字即为 Chat ID。

## 三、系统后台配置

### 1. 启用 Telegram 通知

1. 登录管理员后台，进入 **系统设置** → **通知设置** → **管理员通知**。
2. 确保 **启用管理员通知** 开关已打开。
3. 打开 **Telegram 通知** 开关。

### 2. 填写配置信息

- **Bot Token**：粘贴从 @BotFather 获取的 Bot Token。
- **Chat ID**：粘贴从 @userinfobot 或上述方法获取的 Chat ID。

### 3. 测试通知

1. 点击 **测试 Telegram 通知** 按钮。
2. 系统会发送一条测试消息到您的 Telegram。
3. 如果收到消息，说明配置成功；如果失败，检查 Token 和 Chat ID 是否正确。

### 4. 配置通知事件开关

在 **通知事件开关** 区域，可开启/关闭以下事件的通知：

- **订单支付成功**：用户完成支付时通知
- **新用户注册**：有新用户注册时通知
- **重置密码**：用户重置密码时通知
- **发送订阅**：向用户发送订阅邮件时通知
- **重置订阅**：用户或管理员重置订阅时通知
- **订阅到期**：订阅到期时通知
- **管理员创建用户**：管理员创建新用户时通知
- **订阅创建**：创建新订阅时通知

### 5. 保存配置

点击 **保存管理员通知设置** 按钮保存配置。

## 四、通知消息格式

系统发送的 Telegram 通知包含：

- **标题**：事件类型（如「订单支付成功」「新用户注册」）
- **内容**：详细信息（订单号、用户邮箱、金额等）
- **时间戳**：事件发生时间

示例消息：
```
📦 订单支付成功

订单号: ORD20260210123456
用户: user@example.com
金额: ¥99.00
支付方式: 支付宝
时间: 2026-02-10 14:30:00
```

## 五、常见问题

### Q1: 收不到测试消息？

**检查项：**
1. Bot Token 是否正确（格式：`数字:字母数字组合`）
2. Chat ID 是否正确（纯数字）
3. 是否已向 Bot 发送过至少一条消息（Bot 需要先收到消息才能向您发送）
4. 网络是否正常（服务器能否访问 `api.telegram.org`）

**解决方法：**
- 重新获取 Token 和 Chat ID
- 向 Bot 发送 `/start` 消息
- 检查服务器防火墙是否允许访问 Telegram API

### Q2: 只收到部分通知？

**检查项：**
- 在 **通知事件开关** 中确认对应事件已开启
- 检查系统日志，查看是否有发送失败的记录

### Q3: Bot Token 泄露怎么办？

**处理：**
1. 联系 @BotFather，发送 `/revoke` 撤销当前 Token
2. 重新生成新 Token
3. 在系统后台更新 Bot Token

### Q4: 如何更换接收通知的账号？

**方法：**
1. 使用新账号的 Telegram 联系 @userinfobot 获取新 Chat ID
2. 在系统后台更新 Chat ID
3. 保存配置并测试

## 六、安全建议

1. **保护 Bot Token**：Token 等同于密码，不要泄露或提交到代码仓库
2. **定期检查**：定期测试通知功能，确保正常工作
3. **使用专用 Bot**：建议为系统通知创建专用 Bot，不要与其他用途混用
4. **限制 Bot 权限**：Bot 只需要发送消息权限，不需要其他权限

## 七、高级配置（可选）

### 使用代理（如服务器无法直接访问 Telegram）

如果服务器位于无法直接访问 Telegram API 的地区，可以：

1. 配置 HTTP/HTTPS 代理
2. 在系统环境变量或配置文件中设置代理地址
3. 确保代理可以访问 `api.telegram.org`
