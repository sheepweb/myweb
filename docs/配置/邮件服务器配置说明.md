# 邮件服务器配置说明

## 概述

系统使用 SMTP 发送邮件，用于注册验证、密码重置、订阅通知、到期提醒、工单回复等。可在**环境变量**或**后台邮件设置**中配置 SMTP 与发件人信息。

## 一、配置方式

### 方式一：环境变量（.env）

在项目根目录 `.env` 中配置，供未在后台覆盖时使用：

```env
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USERNAME=your-email@example.com
SMTP_PASSWORD=your-smtp-password
SMTP_FROM_EMAIL=your-email@example.com
SMTP_TLS=true
```

- **SMTP_HOST**：SMTP 服务器地址（如 `smtp.qq.com`、`smtp.163.com`、`smtp.gmail.com`）。
- **SMTP_PORT**：端口，常用 587（TLS）、465（SSL）、25（不加密，不推荐）。
- **SMTP_USERNAME**：登录用户名，多数为完整邮箱。
- **SMTP_PASSWORD**：SMTP 密码或授权码（QQ/163 等需在邮箱设置中开启 SMTP 并获取授权码）。
- **SMTP_FROM_EMAIL**：发件人邮箱，建议与 SMTP_USERNAME 一致。
- **SMTP_TLS**：是否使用 TLS，建议为 true（端口 587 时）。

### 方式二：后台邮件设置

- 路径：**管理员后台** → **系统设置**（或 **设置**）→ **邮件配置** / **通知设置** 等。
- 若存在「邮件」或「SMTP」相关表单，可填写：
  - SMTP 主机、端口、加密方式（TLS/SSL/无）
  - 发件人邮箱、发件人名称
  - SMTP 用户名、SMTP 密码
- 保存后，系统发送邮件时会优先使用后台配置；未配置项仍可能回退到 `.env`。

## 二、常见服务商示例

| 服务商 | SMTP 服务器 | 端口 | 加密 | 说明 |
|--------|-------------|------|------|------|
| QQ 邮箱 | smtp.qq.com | 465/587 | SSL/TLS | 需开启 SMTP 并使用授权码 |
| 163 邮箱 | smtp.163.com | 465/994 | SSL | 需开启 SMTP 并使用授权码 |
| Gmail | smtp.gmail.com | 587 | TLS | 需应用专用密码或 OAuth |
| 阿里云邮箱 | smtp.aliyun.com | 465/25 | 按需 | 按控制台说明配置 |
| SendGrid / 其他 | 按文档 | 587 常见 | TLS | 使用提供的 SMTP 与 API Key/密码 |

## 三、校验与测试

1. 保存配置后，在后台若有「发送测试邮件」按钮，可填写收件人并发送，确认能收到。
2. 触发实际场景：如「忘记密码」发重置邮件、注册验证邮件、订阅通知等，检查收件与发件人显示是否正常。
3. 若发送失败，查看应用日志（如 `server.log` 或 stdout）中的 SMTP 报错（认证失败、连接超时、证书错误等），据此调整主机、端口、加密方式或密码。

## 四、注意事项

- 发件人邮箱建议与 SMTP 登录账号一致，避免被拒信或进垃圾箱。
- 生产环境务必使用 TLS/SSL，避免明文密码传输。
- 部分服务商限制每日发信量，大量通知请考虑专用邮件服务或队列。
- 密码/授权码不要提交到代码仓库，仅放在 `.env` 或后台配置中。
