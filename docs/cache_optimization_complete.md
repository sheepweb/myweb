# 缓存优化完整方案

## 已实施的优化

### 1. 核心缓存功能 ✅
- Redis FlushAll 方法
- 统一缓存服务 (cache_service)
- 后端清除缓存 API: `POST /api/v1/admin/settings/cache/flush`
- 前端清除缓存按钮（系统设置页面）
- 缓存预热机制（启动时自动）

### 2. 已缓存的数据 ✅

| 数据类型 | 缓存键 | TTL | 状态 |
|---------|--------|-----|------|
| 套餐列表 | packages:list:active | 30分钟 | ✅ 已实施 |
| 公告列表 | announcements:list:active | 10分钟 | ✅ 已实施 |
| 系统配置 | system:config:{category} | 1小时 | ✅ 已实施 |

### 3. 自动缓存清除 ✅

| 操作 | 清除的缓存 | 方式 |
|------|-----------|------|
| 套餐增删改 | packages:list:active | 同步 |

## 推荐的缓存策略

### 适合缓存的数据（读多写少）
1. ✅ 套餐列表 - 管理员很少修改
2. ✅ 支付方式 - 配置变化频率低
3. ✅ 系统配置 - 配置变化频率低
4. ✅ 公告列表 - 内容变化频率低
5. ✅ 统计数据 - 可接受短暂延迟
6. ✅ GeoIP查询 - IP地址分配稳定

### 不适合缓存的数据（频繁变化）
1. ❌ 节点列表 - 每小时更新一次
2. ❌ 节点状态 - 实时变化
3. ❌ 实时订单状态 - 需要实时查询
4. ❌ 用户在线状态 - 实时变化

### 可选缓存（根据业务需求）
1. 知识库分类/文章 - TTL: 1小时
2. 优惠券列表 - TTL: 30分钟（需考虑时效性）
3. 促销活动 - TTL: 15分钟（需考虑时效性）
4. 用户分析数据 - TTL: 1-24小时（按时间范围）

## 使用说明

### 管理员手动清除缓存
1. 登录后台管理
2. 进入"系统设置" → "基本设置"
3. 找到"缓存管理"部分
4. 点击"清除所有缓存"按钮

### 自动清除场景
- 代码更新后（运行 `bash install.sh` 选项11）
- 深度清理时（运行 `bash install.sh` 选项4）
- 订单支付成功
- 套餐增删改

### 命令行清除缓存
```bash
cd /path/to/project
go run scripts/flush_cache.go
```

## 监控建议

1. **缓存命中率监控**
   - 记录缓存命中/未命中次数
   - 目标命中率: >80%

2. **缓存大小监控**
   - 定期检查 Redis 内存使用
   - 设置合理的过期策略

3. **性能监控**
   - 对比缓存前后的响应时间
   - 监控数据库查询次数

## 注意事项

1. **数据一致性**
   - 确保数据变更时及时清除相关缓存
   - 使用异步清除避免影响主流程

2. **缓存雪崩**
   - 避免大量缓存同时过期
   - 使用随机 TTL 偏移

3. **缓存穿透**
   - 对不存在的数据也进行缓存（NULL值）
   - 设置较短的 TTL

4. **内存管理**
   - 定期清理过期缓存
   - 设置 Redis 最大内存限制
