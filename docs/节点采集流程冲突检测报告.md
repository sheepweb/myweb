# 节点采集流程冲突检测报告

**检测时间**: 2026-03-05
**检测范围**: 节点采集、转换、缓存清理全流程

---

## 执行流程

```
1. FetchNodesFromURLs
   ↓
2. 下载内容 (HTTP GET)
   ↓
3. TryDecodeNodeList (Base64 解码)
   ↓
4. extractNodeLinks
   ├─→ parseClashYAML (优先)
   │   ├─ YAML 解析成功 → 转换为节点链接 → 返回
   │   └─ YAML 解析失败 → 返回 nil
   │
   └─→ 正则提取 (fallback)
       ├─ 提取所有协议链接
       ├─ 验证链接有效性
       └─ 去重
   ↓
5. processFetchedNodes (过滤关键词)
   ↓
6. deleteAutoImportedNodes (删除旧节点)
   ↓
7. importNodesToDatabaseWithOrder (导入新节点)
   ↓
8. 清除缓存 (异步)
```

---

## 冲突点检测结果

### ✅ 1. YAML 解析 vs 正则提取
**状态**: 无冲突

**原因**:
- YAML 优先执行，只有失败才用正则
- 条件判断: `len(yamlLinks) > 0`
- 不会重复处理同一内容

### ✅ 2. Base64 解码时机
**状态**: 无冲突

**原因**:
- TryDecodeNodeList 在 extractNodeLinks 之前
- 解码后内容传入提取函数
- 不会遗漏编码内容

### ✅ 3. 节点链接去重
**状态**: 无冲突

**原因**:
- 使用 `map[string]bool` 确保唯一性
- 在 extractNodeLinks 返回前统一去重
- 不会产生重复节点

### ✅ 4. 多协议支持
**状态**: 无冲突

**原因**:
- YAML: 支持 ss, vmess, vless, trojan, hysteria2
- 正则: 支持 14+ 种协议
- 协议解析器独立，不会互相干扰

### ✅ 5. 缓存清理时机
**状态**: 无冲突 (已优化)

**原因**:
- 在所有节点导入完成后执行
- 使用 goroutine 异步清理
- 主函数 Sleep 200ms 确保完成
- 清理三种缓存: 节点、系统节点、订阅配置

### ✅ 6. 端口类型转换
**状态**: 无冲突 (已修复)

**原因**:
- getFloat 支持 int, int64, float64, string
- YAML 整数端口正确识别
- 不会出现端口为 0 的情况

### ✅ 7. 节点过滤逻辑
**状态**: 无冲突

**原因**:
- 过滤在提取之后、导入之前
- 不影响原始数据
- 过滤关键词独立配置

### ✅ 8. 手动节点保护
**状态**: 无冲突

**原因**:
- deleteAutoImportedNodes 只删除 IsManual=false
- 手动添加的节点不受影响
- 专线节点得到保护

## 潜在优化建议

### ⚡ 1. 缓存清理改为同步
- **当前**: `go func() { 清理缓存 }()`
- **建议**: 直接同步执行，确保完成后再返回
- **影响**: 低 (200ms 延迟已足够)

### ⚡ 2. YAML 解析错误区分
- **当前**: 解析失败和无节点都返回 nil
- **建议**: 返回 `([]string, error)` 区分错误类型
- **影响**: 低 (当前逻辑已正确处理)

### ⚡ 3. 添加节点转换成功率统计
- **当前**: 只记录失败数量
- **建议**: 记录每种协议的成功/失败数
- **影响**: 无 (仅增强日志)
